## Test Architecture

**Purpose**: Enforce test naming convention compliance across entire codebase

**Test Specifications**:

#### 8.1-UNIT-001a: FunSpec Test Names Must Include Story Reference

**Given**: All test files in project with test() function calls
**When**: Konsist validation executes
**Then**: Each test() call name must match pattern `^\d+\.\d+-[A-Z]+-\d+[a-z]?:.*`

**Implementation**:
```kotlin
test("8.1-UNIT-001a: All FunSpec test() calls must include story reference") {
    val testFiles = Konsist.scopeFromProject()
        .files
        .filter { /* test directories, exclude build/disabled */ }

    testFiles.assertTrue { file ->
        val testCalls = Regex("""test\s*\(\s*"([^"]+)"\s*\)""").findAll(file.text)
        testCalls.all { match ->
            val testName = match.groupValues[1]
            testName.matches(Regex("""^\d+\.\d+-[A-Z]+-\d+[a-z]?:.*"""))
        }
    }
}
```

**Status**: ✅ PASSING

#### 8.1-UNIT-001b: BehaviorSpec Files Must Have Story Reference Comments

**Given**: All BehaviorSpec test files
**When**: Konsist validation executes
**Then**: File must contain story reference comments matching `// Story \d+\.\d+-[A-Z]+-\d+`

**Implementation**:
```kotlin
test("8.1-UNIT-001b: All BehaviorSpec files should have story reference comments") {
    val behaviorSpecFiles = Konsist.scopeFromProject()
        .files
        .filter { /* BehaviorSpec files */ }

    behaviorSpecFiles.assertTrue { file ->
        file.text.contains(Regex("""//\s*Story\s+\d+\.\d+-[A-Z]+-\d+""")) ||
        file.text.contains(Regex("""//\s*\d+\.\d+-[A-Z]+-\d+"""))
    }
}
```

**Status**: ✅ PASSING

---

#### 8.1-UNIT-001c: Test TYPE Values Must Be Valid

**Given**: All test files with story references
**When**: TYPE value extracted from pattern
**Then**: TYPE must be one of: UNIT, INT, E2E

**Implementation**:
```kotlin
test("8.1-UNIT-001c: Test TYPE values must be valid (UNIT, INT, E2E)") {
    val testFiles = Konsist.scopeFromProject().files.filter { /* test files */ }

    testFiles.assertTrue { file ->
        val storyReferences = Regex("""\d+\.\d+-([A-Z]+)-\d+""").findAll(file.text)
        val validTypes = setOf("UNIT", "INT", "E2E")
        storyReferences.all { match ->
            val type = match.groupValues[1]
            validTypes.contains(type)
        }
    }
}
```

**Status**: ✅ PASSING

---

#### 8.1-UNIT-001d: Story References Must Use Valid Format

**Given**: All test files with story references
**When**: EPIC and STORY numbers extracted
**Then**: Both must be numeric (1-99 range)

**Implementation**:
```kotlin
test("8.1-UNIT-001d: Story references must use valid format (EPIC.STORY)") {
    val testFiles = Konsist.scopeFromProject().files.filter { /* test files */ }

    testFiles.assertTrue { file ->
        val storyReferences = Regex("""(\d+)\.(\d+)-[A-Z]+-\d+""").findAll(file.text)
        storyReferences.all { match ->
            val epic = match.groupValues[1].toIntOrNull()
            val story = match.groupValues[2].toIntOrNull()
            epic != null && story != null && epic in 1..99 && story in 1..99
        }
    }
}
```

**Status**: ✅ PASSING

## Edge Cases & Error Scenarios

**Handled Edge Cases**:
1. ✅ Tests in kotlin-disabled/ folders (excluded from validation)
2. ✅ Build artifacts in /bin/ and /build/ (excluded)
3. ✅ Helper files like ProjectFixtures.kt (false positive - excluded as not actual test)
4. ✅ Mixed FunSpec/BehaviorSpec patterns (both supported)
5. ✅ Story ID variations (002a, 002b, 002c for multiple fail-closed tests)

**Error Scenarios**:
- Missing story reference → Konsist test fails with clear error
- Invalid TYPE value → 8.1-UNIT-001c catches
- Invalid format → 8.1-UNIT-001d catches
