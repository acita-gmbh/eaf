### AC8: Graceful Degradation on Nightly Failure

**Validation Test:** 8.7-E2E-008
**Coverage:** **FULL**

**Given:** Nightly pipeline has failed (tests failed or workflow errored)
**When:** Development team attempts to merge PR next day
**Then:**
- PR pipeline executes normally (not affected by nightly failure status)
- Clear separation maintained: PR = blocking quality gate, Nightly = advisory validation
- Nightly failures create GitHub Issues but don't block PR merges
- Issue severity reflects failure type:
  - **INFO:** Property test failures (informational finding)
  - **WARNING:** Mutation coverage regression (quality concern)
  - **CRITICAL:** Fuzz test crash findings (security vulnerability)

**Validation Method:**
1. Manually fail nightly workflow (trigger test failure)
2. Create PR against main branch
3. Verify PR can be merged successfully
4. Verify nightly failure created GitHub Issue

**Test Priority:** P0 (Critical reliability requirement - fail-safe design validation)

**Risk Mitigated:** Validates reliability design philosophy - nightly failures don't disrupt development

**NFR Validation:** Directly validates NFR-REL-001 (nightly failure doesn't block PR merges)
