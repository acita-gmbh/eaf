## Medium Priority Risks

### TECH-001: Convention plugin coverage gaps
**Score: 4 (Medium)**
**Probability**: Medium (2) — New ObservabilityConventionPlugin must be applied consistently across framework and product modules [Source: docs/stories/5.2.implement-prometheus-metrics-micrometer.story.md#tasks--subtasks]
**Impact**: Medium (2) — Missed modules would lead to inconsistent metrics exposure and break AC 1

**Mitigation Strategy**:
- Mirror logging convention rollout pattern to validate plugin application in CI [Source: build-logic/src/main/kotlin/conventions/LoggingConventionPlugin.kt]
- Add smoke tests to confirm registry presence per module
- Document application steps in component specifications (Task 2.4)

**Testing Focus**:
- Build verification across framework/security/product modules
- Gradle task asserting plugin dependency injection
- Integration tests validating registry bean availability

**Owner**: Dev / Build Logic Maintainers
**Timeline**: Prior to final merge of story implementation

## Low Priority Risks

### SEC-001: Metrics endpoint misconfiguration
**Score: 3 (Low)**
**Probability**: Low (1) — Security filter chain already enforces authenticated access patterns [Source: framework/security/src/main/kotlin/com/axians/eaf/framework/security/config/SecurityFilterChainConfiguration.kt]
**Impact**: High (3) — Exposed metrics without auth would leak tenant identifiers and internal topology

**Mitigation Strategy**:
- Enforce `/actuator/prometheus` authentication and validate via Testcontainers [Source: docs/stories/5.2.implement-prometheus-metrics-micrometer.story.md#tasks--subtasks]
- Require `eaf-admin` realm role for scrape operations [Source: shared/testing/src/main/resources/keycloak/eaf-test-realm.json]
- Include security regression tests in Test Design (see test plan)

### PERF-001: Micrometer overhead on hot paths
**Score: 2 (Low)**
**Probability**: Low (1) — Micrometer counters/timers are lightweight, but command/event instrumentation can add latency under load [Source: docs/architecture/components.md#observability-components]
**Impact**: Medium (2) — Elevated latency during command bursts could affect SLAs

**Mitigation Strategy**:
- Adopt async or sample-based metrics for high-volume operations
- Measure baseline via integration tests with synthetic load
- Monitor key meters (`eaf.commands.duration`) post-deployment

### OPS-001: Prometheus scrape integration drift
**Score: 2 (Low)**
**Probability**: Low (1) — Compose stack already exposes Prometheus at port 9090, but configuration drift can occur [Source: docs/architecture/development-workflow.md#services-available]
**Impact**: Medium (2) — Misaligned scrape path would reduce observability coverage for operators

**Mitigation Strategy**:
- Update monitoring documentation and automation to reflect `/actuator/prometheus` path
- Run operational verification tip (curl with JWT) during story QA [Source: docs/stories/5.2.implement-prometheus-metrics-micrometer.story.md#operational-verification-tip]
- Add health check alert in Prometheus configuration (`monitoring/prometheus.yml`)
