### AC 2: Flowable Configured to Use Main PostgreSQL Database with Dedicated `flowable` Schema

#### Scenarios

| ID | Level | Priority | Test Scenario | Justification | Mitigates Risks |
|----|-------|----------|---------------|---------------|-----------------|
| 6.1-INT-003 | Integration | P0 | Query PostgreSQL catalog for `flowable` schema existence | Critical - validates schema isolation strategy | TECH-002 |
| 6.1-INT-004 | Integration | P0 | Verify Flowable uses application DataSource bean (shared DB instance) | Critical - confirms single database requirement | N/A |

**Test Implementation**:
```kotlin
test("Flowable schema should exist in dedicated namespace") {
    val schemaQuery = """
        SELECT table_name FROM information_schema.tables
        WHERE table_schema = 'flowable' AND table_name LIKE 'ACT_%'
        ORDER BY table_name
    """.trimIndent()

    val tables = jdbcTemplate.queryForList(schemaQuery, String::class.java)
    tables.size shouldBeGreaterThan 15 // 15+ Flowable tables expected
    tables.shouldContain("ACT_RU_EXECUTION")
    tables.shouldContain("ACT_RE_DEPLOYMENT")
}
```

class FlowableEngineIntegrationTest : FunSpec() {
    @Autowired

    private lateinit var runtimeService: RuntimeService

    init {

## Test Execution Strategy

### Execution Order

1. **6.1-INT-001**: Dependency resolution (fail fast)
2. **6.1-INT-002**: Convention plugin application
3. **6.1-INT-003**: Schema existence validation (critical)
4. **6.1-INT-004**: DataSource sharing validation (critical)
5. **6.1-INT-005**: Schema tables creation validation (critical)
6. **6.1-INT-007**: Engine beans initialization (critical)
7. **6.1-INT-006**: Operational readiness validation
8. **6.1-INT-008**: BPMN deployment validation

**Rationale**: Build/config tests first (fast failure), then database validation, then engine initialization, finally operational readiness.

### Test Environment Requirements

**Infrastructure**:
- PostgreSQL 16.1+ via Testcontainers
- Spring Boot test context
- Kotest 6.0.3 FunSpec framework

**Configuration**:
- `@SpringBootTest` with RANDOM_PORT
- `@ActiveProfiles("test")`
- `@DynamicPropertySource` for Testcontainers JDBC URL

**Test Data**:
- Minimal BPMN process XML (provided in story)
- No seed data required (infrastructure validation only)
