## Risk-Based Testing Strategy

### Priority 1 – Medium Risks (PERF-001)
- Execute onboarding on representative low-spec machine profile; capture script-reported durations and compare to baseline.

### Priority 2 – Low Risks
- Verify Compose manifest static test executes in CI (`shared/testing` module).
- Confirm `scripts/stop-dev.sh` cleans up resources after manual interrupt.

## Monitoring Recommendations
- Track onboarding duration (available in script summary) and alert if runtime > 20 minutes.
- Surface script exit codes and health-check failures in CI logs for early signal.
- Document `--accept-default-admin` override usage for CI; default flow enforces password capture.

## Residual Risk Summary
- No critical risks remain; only runtime performance requires continued observation when run on constrained hardware.

## Follow-Up Actions
- Track onboarding runtime metrics in CI once Docker runners available.
- Evaluate additional automation (e.g., smoke script) to exercise services post-startup.
