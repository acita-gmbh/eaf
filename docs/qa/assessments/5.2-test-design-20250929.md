## P0 - Critical Test Scenarios

### P0-1: Registry Bean Availability Across Modules
**Type**: Integration Test
**Risk Coverage**: TECH-001
**AC Coverage**: AC 1 (Dependencies managed via convention plugins)

```gherkin
Scenario: Observability convention plugin exposes Prometheus registry in every module
  Given the ObservabilityConventionPlugin is applied to framework and product modules
  When the Spring application context starts for each module type
  Then a single PrometheusMeterRegistry bean is present
  And the bean includes EAF common tags configured by the story
  And no duplicate registry beans are created per module
```

**Validation Checkpoints**
- Application context smoke tests for `framework/observability`, `framework/security`, `products/licensing-server`
- Assert bean existence via ApplicationContextRunner
- Ensure bundle dependencies resolve from `gradle/libs.versions.toml`

### P0-2: `/actuator/prometheus` Requires Authenticated Access
**Type**: Integration Test (Testcontainers)
**Risk Coverage**: SEC-001
**AC Coverage**: AC 2 (Endpoint secured)

```gherkin
Scenario: Prometheus endpoint rejects unauthenticated requests
  Given the licensing-server is running with Keycloak Testcontainer
  When a request without Authorization header hits /actuator/prometheus
  Then the response status is 401 Unauthorized
  And the response body does not leak metrics content
```

**Validation Checkpoints**
- Use Testcontainers to obtain Keycloak URL
- Validate unauthorized request returns 401
- Confirm secure response headers (e.g., `WWW-Authenticate`)

### P0-3: `/actuator/prometheus` Succeeds With `eaf-admin` Token
**Type**: Integration Test (Testcontainers)
**Risk Coverage**: SEC-001, OPS-001
**AC Coverage**: AC 2, AC 4

```gherkin
Scenario: Authorized operator can scrape Prometheus metrics
  Given an access token for the eaf-admin realm role
  When the operator issues GET /actuator/prometheus with Authorization header
  Then the response status is 200 OK
  And the response contains Prometheus exposition format headers
  And the body includes JVM, HTTP server, and Axon metric families
  And metrics are tagged with service_name and tenant_id
```

**Validation Checkpoints**
- Use KeycloakTestTokenProvider to obtain JWT
- Validate presence of `# HELP http_server_requests_seconds` and custom counters
- Confirm tags `service_name="licensing-server"` and `tenant_id` appear

### P0-4: Metrics Tag Integrity Under Tenant Switching
**Type**: Integration Test (TenantContext propagation)
**Risk Coverage**: TECH-001, OPS-001
**AC Coverage**: AC 3, AC 4

```gherkin
Scenario: Tenant context tags each metric emission appropriately
  Given two sequential requests executed under different TenantContext values
  When Micrometer counters are incremented for commands and events
  Then metrics include the correct tenant_id for each request
  And no cross-tenant leakage occurs in exported metrics
```

**Validation Checkpoints**
- Simulate tenant switch via TenantContext utilities [Source: docs/architecture/components.md#tenant-context-management]
- Inspect registry sample outputs for each tenant

### P0-5: Instrumentation Performance Guardrail
**Type**: Performance/Load Test
**Risk Coverage**: PERF-001
**AC Coverage**: AC 3

```gherkin
Scenario: Command throughput remains within SLA after instrumentation
  Given baseline throughput measurements without instrumentation
  When command handlers execute under Micrometer instrumentation for 5 minutes
  Then average latency degradation is <5%
  And no GC/memory spikes occur during the run
```

**Validation Checkpoints**
- Capture baseline vs instrumented metrics
- Monitor `eaf.commands.duration` for latency distribution
- Record GC stats via JFR or Micrometer JVM meters

## P1 - Important Test Scenarios (Highlights)

- **P1-1**: Version Catalog Enforcement — Unit test verifying dependencies pulled from `libs.versions.toml`
- **P1-2**: Registry Auto-configuration Failure Mode — Integration test where Prometheus dependency missing triggers descriptive error
- **P1-3**: Axon Instrumentation Hooks — Integration test ensuring command and event interceptors increment counters appropriately
- **P1-4**: TenantContext Absence — Integration test verifying metrics fail-closed when tenant context unavailable
- **P1-5**: Prometheus `scrape_duration_seconds` validation to ensure exporter stability
- **P1-6**: Operational verification tip automation — E2E script invoking curl with/without JWT and asserting responses
- **P1-7**: Grafana dashboard compatibility smoke test — Ensure metrics families align with expected dashboards

- **P2-1**: YAML configuration propagation for management endpoints
