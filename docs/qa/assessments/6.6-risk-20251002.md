## High Risks Requiring Attention

### TECH-001: BPMN Schema Compliance (Element Ordering)

**Score: 6 (High Risk)**

**Probability**: Medium (2) - Story explicitly warns about this pattern from Story 6.5 lessons

**Impact**: High (3) - Flowable deployment will fail with SAXParseException if violated

**Description**:
BPMN 2.0 XSD schema requires `<documentation>` elements to appear BEFORE `<extensionElements>` in all service tasks. Story 6.5 identified this as a critical implementation blocker when element ordering was violated.

**Affected Components**:
- All BPMN service tasks (PRESCRIPT, CORECOMMAND, POSTSCRIPT, SetCommandType)
- Process-level documentation block

**Mitigation**:
- **Preventive Actions**:
  - Story explicitly documents element ordering requirement in Subtask 1.3
  - Dev Notes § Previous Story Insights warns about this pattern
  - Reference existing BPMN files (hello-world-ansible.bpmn20.xml) for correct patterns
  - Subtask 6.1 specifically validates element ordering

- **Detective Controls**:
  - Integration test validates deployment (catches schema violations immediately)
  - Flowable deployment failure provides clear error message

- **Testing Focus**:
  - 6.6-INT-001: Deploy BPMN and verify successful deployment
  - Manual review of BPMN XML structure before test execution

**Residual Risk**: Low - Pattern is well-documented with test validation catching violations

**Owner**: Dev Agent
**Timeline**: During BPMN template creation (Task 1)
