# Test Design: Story 1.3 "Create One-Command Dev Stack (Docker Compose)"

Date: 2025-09-19
Designer: Quinn (Test Architect)

## Test Strategy Overview

- Total test scenarios: 6
- Unit/Static tests: 1 (17%)
- Integration tests: 4 (66%)
- End-to-end tests: 1 (17%)
- Priority distribution: P0: 3, P1: 2, P2: 1

## Test Scenarios by Acceptance Criteria

### AC1 – Compose manifest exists with required services & versions

| ID              | Level | Priority | Test Focus                                                                 | Justification                                                                                           | Mitigates Risks |
|-----------------|-------|----------|----------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------|-----------------|
| 1.3-UNIT-001    | unit  | P0       | Static validation of `compose.yml` (service names, image tags, ports)     | Can be enforced via YAML parsing/Kotlin test without containers; catches drift quickly                  | TECH-001        |
| 1.3-INT-001     | integration | P0  | `docker compose config`/`docker compose up` dry-run verifying services    | Ensures Compose file renders correctly and includes PostgreSQL, Keycloak, Redis, Prometheus, Grafana     | TECH-001        |

### AC2 – `./scripts/init-dev.sh` boots stack (PostgreSQL, Keycloak) via Compose

| ID              | Level | Priority | Test Focus                                                                 | Justification                                                                                           | Mitigates Risks |
|-----------------|-------|----------|----------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------|-----------------|
| 1.3-INT-002     | integration | P0  | Run `./scripts/init-dev.sh --skip-tests` and assert health endpoints up    | Validates automation flow without long build; verifies containers healthy & network/volume creation     | OPS-001, PERF-001 |
| 1.3-E2E-001     | e2e   | P1       | Full onboarding (no skips) executing quality gates and reporting durations | Confirms Constitutional stack runs end-to-end; captures runtime metrics for performance monitoring      | TEST-001, PERF-001 |

### AC3 – Default credentials, ports, and volumes provisioned for local dev

| ID              | Level | Priority | Test Focus                                                                 | Justification                                                                                           | Mitigates Risks |
|-----------------|-------|----------|----------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------|-----------------|
| 1.3-INT-003     | integration | P0  | Post-onboarding Keycloak realm verification + credential rotation prompt  | Uses Keycloak admin API to confirm realm import and enforces credential change guidance                | SEC-001         |
| 1.3-INT-004     | integration | P2  | Persistent volume regression – rerun onboarding ensuring PostgreSQL data persists or resets deterministically | Validates named volume configuration and documented cleanup commands                                   | DATA-001        |

## Risk Coverage

- **TECH-001**: Covered by 1.3-UNIT-001 and 1.3-INT-001.
- **OPS-001**: Covered by 1.3-INT-002 (startup health) and implicitly validated in 1.3-E2E-001 with teardown assertions.
- **SEC-001**: Covered by 1.3-INT-003 (credential rotation enforcement).
- **TEST-001 / PERF-001**: Covered by 1.3-INT-002 and 1.3-E2E-001 measuring runtime and offering `--skip-tests` guardrail.
- **DATA-001**: Covered by 1.3-INT-004 verifying volume behavior.

## Recommended Execution Order

1. 1.3-UNIT-001 – fast static validation to catch manifest drift.
2. 1.3-INT-001 – verify Compose structure before running full script.
3. 1.3-INT-002 – bring stack up with `--skip-tests` for quick sanity.
4. 1.3-INT-003 – check Keycloak realm and credential workflow.
5. 1.3-INT-004 – regression on persistent volumes.
6. 1.3-E2E-001 – full Constitutional onboarding, record runtime metrics.
