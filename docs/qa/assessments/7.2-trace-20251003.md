##### 7.2-UNIT-006: Security Attack Scenarios

**Test File**: `tools/eaf-cli/src/test/kotlin/com/axians/eaf/tools/cli/commands/ModuleCommandTest.kt`
**Test Level**: Unit
**Priority**: P0 (SECURITY CRITICAL)

**Attack Scenario 1: Path Traversal**

**Given**: Module name "../../etc/passwd" attempting directory escape
**When**: Validation regex is applied
**Then**:
- Validation fails (regex `^[a-z][a-z0-9-]*$` rejects `..` and `/`)
- Security violation error message
- No file system operations attempted

**Attack Scenario 2: Template Injection**

**Given**: Module name "<script>alert('xss')</script>" attempting HTML injection
**When**: Validation logic checks for `<` and `>` characters
**Then**:
- Validation fails (< and > characters rejected)
- Security violation error message
- Template engine never receives malicious input

**Attack Scenario 3: Command Injection**

**Given**: Module name "module; rm -rf /" attempting shell injection
**When**: Validation regex is applied
**Then**:
- Validation fails (semicolon not in `[a-z0-9-]`)
- Security violation error message
- No shell commands executed

**Attack Scenario 4: Null Byte Injection**

**Given**: Module name "module\0.txt" attempting null byte attack
**When**: Validation regex is applied
**Then**:
- Validation fails (null byte not printable ASCII)
- Security violation error message
- File system receives safe input only

**Attack Scenario 5: Unicode Normalization**

**Given**: Module name "\u202Emodule" with right-to-left override character
**When**: Validation regex is applied to Unicode string
**Then**:
- Validation fails (non-ASCII character)
- Security violation error message
- No Unicode exploits possible

**Coverage Aspect**: Defense-in-depth security validation (SEC-002 risk mitigation)

**Security Principle**: Fail-fast validation prevents downstream exploitation
