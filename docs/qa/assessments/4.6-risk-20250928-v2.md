**Testing Requirements**:

**Residual Risk**: Low - Runtime failures are deterministic and debuggable

### Low Priority Risks (Score: 2)

#### IMPL-001: Pattern Conversion Introduces Bugs (NEW RISK)
**Score: 2 (Low)**
**Category**: Technical Implementation
**Probability**: Low (1) - Pattern is straightforward, proven in framework modules
**Impact**: Medium (2) - Could delay story but easily detected and fixed

**Affected Components**:
- All 5 test files being converted
- Dependency injection wiring
- Test lifecycle hooks (beforeEach, afterEach)
- TestContainers listeners

**Rationale**:
- Conversion requires manual refactoring of 5 test files (1,400+ lines total)
- Risk of copy-paste errors, missing @Autowired annotations, incorrect init block structure
- Could miss lifecycle hooks (beforeSpec, afterSpec)
- Might incorrectly move companion object code into init block

**Mitigation Strategy** (Preventive):
1. **Use Template**: Follow exact pattern from framework/security/TenantContextFilterIntegrationTest.kt
2. **Checklist Per Test**:
   - ✅ Remove constructor parameters
   - ✅ Add @Autowired fields for each dependency
   - ✅ Change `FunSpec({...})` to `FunSpec() { init {...} }`
   - ✅ Move test code into init block
   - ✅ Keep companion object with @DynamicPropertySource
   - ✅ Add import: `org.springframework.beans.factory.annotation.Autowired`
3. **Compile After Each**: Verify compilation after each test conversion
4. **Peer Pattern Check**: Compare converted test structure with working framework test
5. **Git Tracking**: Commit after each successful conversion for rollback safety

**Testing Requirements**:
- Compile after each test conversion
- Visual diff against framework/security test pattern
- Run converted test individually before proceeding to next

**Residual Risk**: Very Low - Compilation errors immediately visible, pattern is simple

**Owner**: Dev Agent
**Timeline**: Task 2 (during pattern conversion)

**Common Conversion Errors to Watch**:
- Forgetting `@Autowired` annotation
- Forgetting to add `Autowired` import
- Incorrect init block syntax
- Moving companion object inside init block (wrong!)
- Missing `extension(SpringExtension())` call
