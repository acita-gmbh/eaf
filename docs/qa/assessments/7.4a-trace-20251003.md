## Edge Cases Coverage

### Edge Case 1: JWT without tenant_id

**Test**: 7.4a-UNIT-P0-003 (Tenant Context Propagation)

**Scenario**:
- **Given**: JWT missing `tenant_id` claim (Keycloak misconfiguration)
- **When**: `dataProvider` attempts API request
- **Then**: Request rejected with error (fail-closed behavior)

**Risk Mitigated**: DATA-001 (Keycloak configuration error)

### Edge Case 2: Expired refresh token

**Test**: 7.4a-UNIT-P0-006 (Token Refresh Edge Cases)

**Scenario**:
- **Given**: Valid access token, expired refresh token
- **When**: `authProvider.checkAuth()` attempts token refresh
- **Then**: Method rejects, triggers full re-login flow

**Risk Mitigated**: SEC-003 (authentication flow errors)

### Edge Case 3: Malformed RFC 7807 response

**Test**: 7.4a-UNIT-P1-003 (RFC 7807 Error Mapping)

**Scenario**:
- **Given**: API response with non-RFC 7807 error structure (missing fields)
- **When**: `dataProvider` error mapper processes response
- **Then**: Generic error message provided, raw error logged (no crash)

**Risk Mitigated**: TECH-002 (error mapping incompleteness)

### Edge Case 4: Network error during token refresh

**Test**: 7.4a-UNIT-P0-006 (Token Refresh Edge Cases)

**Scenario**:
- **Given**: Network error during `refreshAccessToken` call
- **When**: `authProvider.checkAuth()` attempts token refresh
- **Then**: Method rejects, triggers re-login flow (graceful degradation)

**Risk Mitigated**: SEC-003 (authentication flow errors), Reliability NFR
