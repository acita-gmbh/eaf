## Test Execution Phases

### Phase 1: Root Cause Investigation (1-3 hours)
**Purpose**: Systematically identify the root cause using 5 hypotheses

**Tests**: 9.2-MAN-001 through 9.2-MAN-006 (6 manual tests)
1. Enable debug logging (9.2-MAN-001)
2. Verify QueryHandler registration (9.2-MAN-002)
3. Verify interceptor timing (9.2-MAN-003)
4. Verify session variable (9.2-MAN-004)
5. Test RLS isolation (9.2-MAN-005)
6. Test repository directly (9.2-MAN-006)

**Success Criteria**: Root cause identified from hypotheses 1-5

---

### Phase 2: Fix Validation (0.5-1 hour)
**Purpose**: Verify applied fix resolves the issue

**Tests**: 9.2-INT-001 through 9.2-INT-005 (5 integration tests)
1. QueryGateway execution (9.2-INT-001) - **Most critical**
2. API response validation (9.2-INT-002)
3. Pagination headers (9.2-INT-003)
4. Interceptor automated test (9.2-INT-004)
5. RLS enforcement test (9.2-INT-005) - **Security critical**

**Success Criteria**: All integration tests pass, no ExecutionException

---

### Phase 3: End-to-End Validation (1-2 hours)
**Purpose**: Validate full user flow and unblock Story 9.1

**Tests**: 9.2-E2E-001 through 9.2-MAN-007 (4 E2E/manual tests)
1. Frontend empty state (9.2-E2E-001)
2. Frontend shows error resolved (9.2-E2E-002)
3. Create + list full flow (9.2-E2E-003)
4. Story 9.1 Tasks 5-7 validation (9.2-MAN-007)

**Success Criteria**: Story 9.1 validation complete, all 41 ACs validated

---

### Phase 4: Documentation (30 minutes)
**Purpose**: Capture knowledge for future reference

**Tests**: 9.2-MAN-008 (1 manual test)
1. Document root cause, fix, and lessons learned

**Success Criteria**: Dev Agent Record updated with comprehensive notes
