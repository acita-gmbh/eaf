## Special Test Considerations

### Memory Management Testing
- **WeakReference Validation**: Force garbage collection scenarios
- **ThreadLocal Cleanup**: Verify complete removal from pooled threads
- **Stack Depth Monitoring**: Validate production alerting mechanisms
- **Extended Runs**: Long-running tests to detect gradual memory leaks

### Security Testing
- **JWT Manipulation**: Various malformed claim scenarios
- **Tenant Isolation**: Cross-tenant access attempts
- **Fail-Closed Validation**: All security boundary enforcement
- **DoS Resilience**: High-frequency switching patterns

### Performance Testing
- **Filter Overhead**: <5ms processing time validation
- **Concurrent Load**: 100+ concurrent request handling
- **Memory Usage**: ThreadLocal memory footprint analysis
- **Stack Depth Impact**: Performance with varying stack depths

### Production Readiness Testing
- **Pooled Thread Safety**: Application server thread pool scenarios
- **Exception Handling**: All error path cleanup validation
- **Monitoring Integration**: Alerting and metrics validation
- **Cleanup Verification**: Zero-leak guarantee testing

- **Framework**: Kotest 6.0.3 (FunSpec)

- **Framework**: Kotest + Testcontainers
