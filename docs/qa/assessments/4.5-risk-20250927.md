## Critical Risks Requiring Immediate Attention

### TECH-001: Build Dependency Mismatch

**Score: 9 (Critical)**
**Probability**: High (3) - Many modules touch widget code, complex dependency graph
**Impact**: High (3) - Entire build can fail and block all downstream work

**Description**: Removing the `framework:widget` dependency and adding product-module dependency can leave orphaned transitive dependencies or duplicate classes, resulting in compile-or-runtime errors.

**Affected Components**:
- `build.gradle.kts` (all projects)
- All modules that depend on `products/widget-demo`

**Mitigation Strategy**:
- Re-run full multi-module clean build and capture dependency reports
- Validate that `products/widget-demo` is sole provider of Widget.* classes
- Verify no unused "old" transitive dependencies remain

**Testing Requirements**:
- `./gradlew clean build --scan --info` execution
- Dependency-tree inspection: `./gradlew dependencies`
- Build cache verification to ensure no stale artifacts

### TECH-003: Spring Modulith Component-Scan Breakage

**Score: 6 (High)**
**Probability**: Medium (2) - Depends on config-file location and scan settings
**Impact**: High (3) - Application fails at startup for end-to-end scenarios

**Description**: Spring-Modulith uses component-scan boundaries defined by module's base package. Moving package to `products.widgetdemo` may cause `@SpringBootApplication` or `@EnableModule` config to miss new components, leading to `BeanNotFound` errors.

**Affected Components**:
- `WidgetModule.kt`
- Spring configuration
- `@Component`, `@Service`, `@Repository`, `@EventHandler`, `@QueryHandler` classes

**Mitigation**:
- Verify scan base package is updated
- Add module-boundary test that instantiates Spring context

**Testing**:
- `./gradlew :products:widget-demo:test` (includes context load tests)
- Application-context load integration test (`ApplicationContextLoads`)

**Code-Specific Insight** (consult7):
- `WidgetModule.kt` uses `@ApplicationModule(allowedDependencies = ["core", "shared::api", "shared::testing"])`
- Must update to include framework dependencies: `["core", "security", "cqrs", "persistence", "web", "shared.api"]`

**Impact**: High (3) - Core business functions may stop reacting

**Affected Components**:

- `WidgetQueryHandler`

**Mitigation**:

- Verify event-handler registration via `applicationContext.getBeansOfType(EventHandlerAspect)`

**Testing**:
- Spring Boot integration test publishing `WidgetCreatedEvent` and asserting `WidgetProjection` update
- Axon Test Fixtures (`EventBus`) to verify handler invocation

**Code-Specific Insight** (consult7):
- `WidgetProjectionHandler.kt` uses `@ProcessingGroup("widget-projection")`
- Any Axon configuration targeting this processing group by package name must be updated

## Code-Specific Risks (consult7 Analysis)

### CRITICAL: Event Sourcing Serialization Risk

**Finding**: Widget domain uses Axon Framework event sourcing with events stored in event store using fully qualified class names.

**Risk**: If `com.axians.eaf.api.widget.*` package is moved or renamed, existing events in event store will fail to deserialize.

**Mitigation** (Story correctly identifies):
- **Shared API Boundary**: `com.axians.eaf.api.widget.*` must NOT move
- Commands, Events, Queries remain in `shared/shared-api/`
- Only IMPLEMENTATION (aggregate, handlers) moves to products

**Status**: ✅ MITIGATED by story design

### Package Dependencies Analysis

**External Dependencies That Must Remain Accessible**:
1. `com.axians.eaf.api.widget.*` - Commands, Events, Queries (MUST NOT MOVE)
2. `com.axians.eaf.framework.persistence.*` - WidgetProjection entity and repository
3. `com.axians.eaf.framework.security.tenant.TenantContext` - Tenant isolation

**Risk**: If framework packages move, imports will break

**Mitigation**: Story correctly identifies framework modules stay in place; only widget domain moves

**Status**: ✅ ADDRESSED in migration plan

### Spring Modulith Configuration

**Finding**: `WidgetModule.kt` declares `@ApplicationModule(allowedDependencies = ["core", "shared::api", "shared::testing"])`

**Risk**: Insufficient dependencies declared for product module

**Required Dependencies**: `["core", "security", "cqrs", "persistence", "web", "shared.api"]`

**Status**: ✅ ADDRESSED in Subtask 5.2

### Axon Processing Group

**Finding**: `@ProcessingGroup("widget-projection")` in WidgetProjectionHandler

**Risk**: Axon configuration in `application.yml` targeting this processing group by package name

**Mitigation**: Verify Axon configuration is package-agnostic (uses string literal)

**Status**: ⚠️ SHOULD VERIFY - Add to validation checklist
