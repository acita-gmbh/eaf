## Requirements to Tests - Given/When/Then Format

### Critical Path Tests

**8.2-INT-002: Hook Blocks Violations**
- **Given**: Staged Kotlin file with ktlint format violations
- **When**: Developer executes `git commit -m "test"`
- **Then**:
  - Pre-commit hook executes
  - ktlint check fails
  - Commit is blocked (exit code 1)
  - Error message shows which files violated
  - Fix command displayed: `./gradlew ktlintFormat`
- **Validates**: ACs 5, 12, 13

**8.2-INT-003: Hook Allows Clean Commit**
- **Given**: Staged files with no violations
- **When**: Developer executes `git commit -m "[EPIC-8] feat: add pre-commit hooks"`
- **Then**:
  - All checks pass (format, static analysis, test naming, unit tests, commit message)
  - Progress output shown with timings
  - Total execution time <30s
  - Commit succeeds
  - Success message displayed
- **Validates**: ACs 5, 6, 7, 8, 10, 12

**8.2-INT-005: Execution Within Timeout**
- **Given**: Large commit (10 modules, 50 files)
- **When**: Pre-commit hook executes
- **Then**:
  - Execution completes OR times out at 30s
  - If timeout: graceful termination with message
  - Developer can proceed with --no-verify if needed
- **Validates**: AC 8, Performance target

**8.2-INT-006: Cross-Platform Compatibility**
- **Given**: Repository cloned on Windows, Mac, Linux
- **When**: Hooks installed and commit attempted on each platform
- **Then**:
  - Installation succeeds on all 3 platforms
  - Validation executes correctly on all platforms
  - Exit codes and error messages consistent
  - No platform-specific failures
- **Validates**: Reliability NFR, RISK-002 mitigation
