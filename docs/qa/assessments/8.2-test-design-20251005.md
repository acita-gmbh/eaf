# Test Design: Story 8.2 - Pre-Commit Hook Infrastructure

**Story**: 8.2 - Establish Pre-Commit Hook Infrastructure
**Assessment Date**: 2025-10-05
**Assessor**: Quinn (Test Architect)
**Status**: PRE-IMPLEMENTATION

---

## Test Strategy Summary

**Testing Approach**: Meta-Testing (Testing the hooks themselves)
**Framework**: Kotest 6.0.3 + Shell script testing
**Test Levels**: 4 (Installation, Validation Logic, Integration, Performance)
**Estimated Test Count**: ~25 tests
**Test Execution Target**: Hooks themselves must be fast (<30s), hook tests can be slower

---

## Test Architecture

### Test Level 1: Hook Installation Tests (P0 - Critical)

**Purpose**: Validate hooks install correctly and are executable

#### 8.2-UNIT-001: installGitHooks Gradle task creates hook files

**Given**: Clean repository with no existing git hooks
**When**: `./gradlew installGitHooks` executes
**Then**:
- `.git/hooks/pre-commit` file created
- `.git/hooks/commit-msg` file created
- Both files have executable permissions (chmod +x)

**Test Type**: Unit (Gradle task testing)
**Priority**: P0 (Critical)

---

#### 8.2-UNIT-002: Pre-commit hook file is executable

**Given**: Hook installed via installGitHooks task
**When**: File permissions checked
**Then**: File has executable bit set (can run via `./git/hooks/pre-commit`)

**Test Type**: Unit
**Priority**: P0

---

#### 8.2-UNIT-003: Commit-msg hook file is executable

**Given**: Hook installed via installGitHooks task
**When**: File permissions checked
**Then**: File has executable bit set

**Test Type**: Unit
**Priority**: P0

---

#### 8.2-INT-001: Hooks auto-install during init-dev.sh execution

**Given**: Fresh repository clone
**When**: `./scripts/init-dev.sh` executes
**Then**:
- Hooks installed automatically
- Developer sees "Installing pre-commit hooks..." message
- Hooks functional immediately

**Test Type**: Integration (script testing)
**Priority**: P0

---

### Test Level 2: Validation Logic Tests (P1 - Important)

**Purpose**: Test each validation check independently

#### 8.2-UNIT-004: ktlint validation detects formatting violations

**Given**: Staged Kotlin file with format violations
**When**: ktlint check runs on staged files
**Then**:
- Violations detected
- Specific files and line numbers reported
- Exit code non-zero

**Test Type**: Unit
**Priority**: P1

---

#### 8.2-UNIT-005: Detekt validation runs on staged files only

**Given**: 2 Kotlin files changed, 1 staged, 1 unstaged
**When**: Detekt pre-commit check runs
**Then**:
- Only staged file analyzed
- Unstaged file ignored
- Fast execution (<8s)

**Test Type**: Unit
**Priority**: P1

---

#### 8.2-UNIT-006: Konsist test naming validation integrates Story 8.1

**Given**: Staged test file with missing story reference
**When**: Konsist naming validation runs
**Then**:
- TestCaseNamingConventionTest from Story 8.1 executes
- Violation detected
- Clear error message referencing naming standard

**Test Type**: Integration (Story 8.1 dependency)
**Priority**: P0 (validates Story 8.1 integration)

---

#### 8.2-UNIT-007: Smart module detection from staged files

**Given**: Files staged in 3 different modules
**When**: Module detection script runs
**Then**:
- All 3 modules identified correctly
- Only those modules' unit tests will run
- Other modules skipped

**Test Type**: Unit
**Priority**: P1

---

#### 8.2-UNIT-008: Commit message regex validation

**Given**: Various commit message formats
**When**: Validation regex executes
**Then**:
- Valid: `[DPCMSG-1234] fix: description` → ✅ Pass
- Valid: `[Epic 8] test: description` → ✅ Pass
- Invalid: `updated files` → ❌ Fail
- Invalid: `[WIP] testing` → ⚠️ Bypass allowed

**Test Type**: Unit
**Priority**: P1

---

### Test Level 3: Integration Tests (P0 - Critical)

**Purpose**: Validate end-to-end hook execution

#### 8.2-INT-002: Pre-commit hook blocks commit with violations

**Given**: Staged file with ktlint violations
**When**: `git commit -m "test"` executes
**Then**:
- Pre-commit hook runs
- Violations detected
- Commit blocked (exit 1)
- Helpful error message displayed

**Test Type**: End-to-End
**Priority**: P0

---

#### 8.2-INT-003: Pre-commit hook allows clean commit

**Given**: Staged files with no violations
**When**: `git commit -m "[EPIC-1] feat: test"` executes
**Then**:
- All checks pass
- Commit succeeds
- Execution time <30s
- Success message displayed

**Test Type**: End-to-End
**Priority**: P0

---

#### 8.2-INT-004: --no-verify bypass works correctly

**Given**: Staged file with violations
**When**: `git commit --no-verify -m "test"` executes
**Then**:
- Hooks skipped
- Commit succeeds despite violations
- Bypass logged to metrics (for tracking)

**Test Type**: Integration
**Priority**: P0

---

#### 8.2-INT-005: Hook execution completes within 30s timeout

**Given**: Maximum realistic commit (10 modules changed, 50 files)
**When**: Pre-commit hook runs
**Then**:
- Execution completes within 30s
- If timeout exceeded, graceful termination
- Clear timeout message displayed

**Test Type**: Performance / Integration
**Priority**: P0

---

#### 8.2-INT-006: Hooks work on Windows/Mac/Linux

**Given**: Same repository cloned on 3 platforms
**When**: Hooks installed and commit attempted
**Then**:
- Installation succeeds on all platforms
- Validation executes correctly on all platforms
- Exit codes consistent across platforms

**Test Type**: Cross-Platform Integration
**Priority**: P0 (validates RISK-002 mitigation)

**Implementation**:
- CI matrix build with windows-latest, macos-latest, ubuntu-latest
- Test hook installation and execution on each platform

---

### Test Level 4: Performance Tests (P1 - Important)

#### 8.2-PERF-001: Total hook execution < 30s (p95)

**Given**: 100 sample commits with varying sizes
**When**: Pre-commit hooks execute for each
**Then**:
- p50 execution time <20s
- p95 execution time <30s
- p99 execution time <35s
- Mean execution time <18s

**Test Type**: Performance Benchmark
**Priority**: P1

---

#### 8.2-PERF-002: ktlint validation < 5s

**Given**: 20 staged Kotlin files (typical commit size)
**When**: ktlint check executes
**Then**: Execution time <5s (p95)

**Test Type**: Performance Benchmark
**Priority**: P2

---

#### 8.2-PERF-003: Unit tests < 20s (changed modules only)

**Given**: Changes in 2 modules (e.g., framework/security + products/widget-demo)
**When**: Smart test execution runs
**Then**:
- Only 2 modules' unit tests execute
- Total time <20s
- Other modules skipped

**Test Type**: Performance Benchmark
**Priority**: P1

---

### Test Level 5: Developer Experience Tests (P1)

#### 8.2-E2E-001: Error messages include fix commands

**Given**: Commit with ktlint violations
**When**: Hook fails
**Then**: Error output includes:
- ✅ What failed (ktlint format violations)
- ✅ Which files affected
- ✅ How to fix (`./gradlew ktlintFormat`)
- ✅ How to bypass (`--no-verify`)

**Test Type**: User Experience
**Priority**: P1

---

#### 8.2-E2E-002: Progress output shows time per check

**Given**: Clean commit
**When**: Hook executes successfully
**Then**: Each check shows timing:
- ✅ "Format checking...✓ (2.1s)"
- ✅ "Static analysis...✓ (3.4s)"
- ✅ Total time displayed

**Test Type**: User Experience
**Priority**: P2

---

#### 8.2-E2E-003: Metrics logged correctly

**Given**: 10 commits with hooks
**When**: Metrics reviewed
**Then**: `.git/hooks/metrics.log` contains:
- Timestamp per commit
- Execution time per check
- Total execution time
- Pass/fail status

**Test Type**: Metrics Validation
**Priority**: P2

## Test Execution Strategy

**Pre-Rollout Testing**:
1. Unit tests for all validation logic
2. Integration tests on developer machine
3. Cross-platform testing in CI matrix
4. Performance benchmarks with realistic loads

**During Phased Rollout**:
- Week 1: Monitor false positives (adjust rules)
- Week 2: Monitor performance (optimize if needed)
- Week 3: Monitor adoption rate (improve UX if low)

**Post-Rollout**:
- Continuous metrics collection (AC 18)
- Quarterly test suite review
- Add tests for new validation rules
