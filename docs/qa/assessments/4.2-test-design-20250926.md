# Test Design: Story 4.2 - Layer 2 Service Tenant Validation

**Story**: 4.2 - Implement Layer 2 (Service Layer): Tenant Boundary Validation
**Date**: 2025-09-26
**QA Reviewer**: Quinn (Test Architect)
**Assessment Type**: Comprehensive Test Scenario Design

## Executive Summary

This document defines comprehensive test scenarios for Story 4.2 tenant validation implementation. Test design follows risk-based prioritization with **24 unit test scenarios** and **12 integration test scenarios** organized by priority level.

**Test Coverage Strategy**:
- **Unit Tests (60%)**: Fast feedback, tenant validation logic isolation
- **Integration Tests (40%)**: Cross-tenant attack scenarios, transaction rollback verification
- **Performance Tests (10%)**: Latency baseline, concurrent load validation

**Risk-Based Prioritization**:
- **P0 (Must Have)**: 18 scenarios - Security-critical tenant isolation validation
- **P1 (Should Have)**: 12 scenarios - Transaction rollback, error handling consistency
- **P2 (Nice to Have)**: 6 scenarios - Edge cases, performance monitoring

**Total Test Scenarios**: 36 scenarios
**Estimated Implementation**: 8-12 hours (unit tests), 12-16 hours (integration tests)

#### UT-4.2-009: Command Tenant Mismatch - Either.left (P0)
**Priority**: P0 (MUST HAVE)
**Risk**: 5.6/10 (Critical security control)
**Test Framework**: Kotest BehaviorSpec
**Duration**: <1ms

**Given**:
- TenantContext set to "tenant-a"
- Existing Widget with tenantId="tenant-a"
- UpdateWidgetCommand with tenantId="tenant-b"

**When**:
- Widget.handle(UpdateWidgetCommand) invoked

**Then**:
- Either.left(WidgetError.TenantIsolationViolation) returned
- Error includes requestedTenant="tenant-b", actualTenant="tenant-a"
- Widget state UNCHANGED

**Test Code**:
```kotlin
Given("an existing Widget for tenant-a") {
    tenantContext.setCurrentTenantId("tenant-a")

    val createCommand = CreateWidgetCommand(
        widgetId = "widget-update-2",
        tenantId = "tenant-a",
        name = "Original Name"
    )

    val widget = Widget(createCommand)

    When("UpdateWidgetCommand with mismatched tenant") {
        val updateCommand = UpdateWidgetCommand(
            widgetId = "widget-update-2",
            tenantId = "tenant-b",  // Mismatch!
            name = "Hacked Name"
        )

        val result = widget.handle(updateCommand)

        Then("update should be rejected") {
            result.shouldBeLeft()

            val error = result.leftValue
            error.shouldBeInstanceOf<WidgetError.TenantIsolationViolation>()
            error.requestedTenant shouldBe "tenant-b"
            error.actualTenant shouldBe "tenant-a"
        }

        Then("widget state should be unchanged") {
            widget.name shouldBe "Original Name"  // NOT "Hacked Name"
        }
    }
}
```

**Validation**:
- [ ] Either.left returned
- [ ] TenantIsolationViolation error
- [ ] Widget state unchanged

### 2.1 Cross-Tenant Attack Scenarios (6 scenarios)

#### IT-4.2-001: Cross-Tenant Create Attack (P0)
**Priority**: P0 (MUST HAVE)
**Risk**: 6.0/10 (Tenant isolation bypass)
**Test Framework**: Kotest BehaviorSpec with Spring Boot Test
**Duration**: ~500ms

**Given**:
- TenantContext set to "tenant-a" (valid JWT)
- User authenticated as Tenant A

**When**:
- CreateWidgetCommand with tenantId="tenant-b" sent via CommandGateway

**Then**:
- Command rejected with SecurityException or 403 Forbidden
- No widget created for Tenant B
- No events persisted in event store

**Test Code**:
```kotlin
@SpringBootTest
@Testcontainers
class TenantBoundaryValidationIntegrationTest : BehaviorSpec() {

    companion object {
        @Container
        val postgres = PostgreSQLContainer("postgres:16.1-alpine")

        @Container
        val axonServer = GenericContainer("axoniq/axonserver:latest")
            .withExposedPorts(8024, 8124)
    }

    @Autowired
    private lateinit var commandGateway: CommandGateway

    @Autowired
    private lateinit var tenantContext: TenantContext

    @Autowired
    private lateinit var eventStore: EventStore

    init {
        Given("User authenticated as Tenant A") {
            tenantContext.setCurrentTenantId("tenant-a")

            When("CreateWidgetCommand sent for Tenant B") {
                val command = CreateWidgetCommand(
                    widgetId = "widget-attack-1",
                    tenantId = "tenant-b",  // Cross-tenant attack
                    name = "Malicious Widget"
                )

                Then("command should be rejected") {
                    shouldThrow<Exception> {
                        commandGateway.sendAndWait<String>(command)
                    }.message shouldContain "Tenant isolation violation"
                }

                Then("no events should be persisted") {
                    val events = eventStore.readEvents("widget-attack-1").asStream().toList()
                    events shouldBe emptyList()
                }
            }

            tenantContext.clearCurrentTenant()
        }
    }
}
```

**Validation**:
- [ ] Command rejected
- [ ] No events in event store
- [ ] SecurityException or 403 status

## 5. Test Anti-Patterns to Avoid

### ❌ PROHIBITED

1. **Mocking TenantContext**:
   ```kotlin
   // ❌ FORBIDDEN
   val mockContext = mockk<TenantContext>()
   every { mockContext.getCurrentTenantId() } returns "tenant-a"
   ```

2. **Mocking Aggregates**:
   ```kotlin
   // ❌ FORBIDDEN
   val mockWidget = mockk<Widget>()
   every { mockWidget.handle(any()) } returns Either.right(Unit)
   ```

3. **Using H2 in Integration Tests**:
   ```kotlin
   // ❌ FORBIDDEN
   @DataJpaTest  // Uses H2 by default
   ```

4. **Mixing JUnit and Kotest**:
   ```kotlin
   // ❌ FORBIDDEN
   class WidgetTest : FunSpec({
       @Test  // ← This annotation is IGNORED by Kotest
       fun testWidget() { }
   })
   ```

### ✅ REQUIRED PATTERNS

1. **Real TenantContext**:
   ```kotlin
   val tenantContext = TenantContext(SimpleMeterRegistry())
   ```

2. **Testcontainers for Integration Tests**:
   ```kotlin
   @Container
   val postgres = PostgreSQLContainer("postgres:16.1-alpine")
   ```

3. **Kotest afterTest Hook**:
   ```kotlin
   afterTest {
       tenantContext.clearCurrentTenant()
   }
   ```
