### CRITICAL: 8.3-REPO-001 - Tenant Boundary Violation in save()

**Vulnerable Code:**
```kotlin
override fun save(projection: WidgetProjection): WidgetProjection {
    val record = projection.toRecord()

    val saved = dsl
        .insertInto(WidgetProjectionTable.WIDGET_PROJECTION)
        .set(record)
        .onConflict(WidgetProjectionTable.WIDGET_PROJECTION.WIDGET_ID)
        .doUpdate()
        .set(WidgetProjectionTable.WIDGET_PROJECTION.TENANT_ID, record.tenantId) // ⚠️ SECURITY ISSUE
        // ... other fields
        .returning()
        .fetchOne() ?: error("Failed to persist widget projection ${projection.widgetId}")

    return saved.toDomain()
}
```

**Attack Scenario:**
1. Tenant A creates widget with `widget_id = "abc123"`
2. Tenant B calls `save()` with same `widget_id` but different `tenant_id`
