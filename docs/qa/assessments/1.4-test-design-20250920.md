# Test Design: Story 1.4 Establish Foundational CI Pipeline

- **Date:** 2025-09-20
- **Designer:** Quinn (Test Architect)
- **Story:** docs/stories/1.4.establish-foundational-ci-pipeline.story.md
- **Acceptance Criteria:** Epic 1 Story 1.4 (AC1–AC4)
- **Linked Risk Profile:** docs/qa/assessments/1.4-risk-20250920.md

## Test Strategy Overview
- Total test scenarios: 6
- By level: Unit 1 (17%), Integration 3 (50%), E2E 2 (33%)
- By priority: P0 = 4, P1 = 2, P2 = 0, P3 = 0

Execution order: Run `actionlint` unit check first, followed by P0 integration suites (`quality-gates`, `build`, `integrationTest`), then enforce branch protection E2E validation and finally runtime/performance sampling.

## Test Scenarios by Acceptance Criteria

### AC1 – Pipeline triggers automatically on PRs to `main`

| ID | Level | Priority | Test | Justification | Mitigates |
|----|-------|----------|------|---------------|-----------|
| 1.4-E2E-001 | E2E | P0 | Simulate PR to `main` using `gh workflow run`/`act pull_request` and confirm `Continuous Integration` workflow dispatches automatically. | Directly validates event triggers per AC1. | OPS-001 |

### AC2 – Execute compile + quality gates (`compile`, `check`, `test`)

| ID | Level | Priority | Test | Justification | Mitigates |
|----|-------|----------|------|---------------|-----------|
| 1.4-UNIT-001 | Unit | P1 | Run `actionlint` against `.github/workflows/ci.yml` to ensure workflow syntax and step ordering (including compile/build step) stay intact. | Fast feedback when editing workflow; prevents regressions before CI run. | TECH-001 |
| 1.4-INT-001 | Integration | P0 | Execute `quality-gates` job on hosted runner; assert `ktlintCheck`, `detekt`, `konsistTest` tasks run and fail on injected violation. | Guarantees zero-violation enforcement mandated by architecture. | SEC-001 |
| 1.4-INT-002 | Integration | P0 | Run `build` job (`./gradlew build`) and verify `compileKotlin`, `test`, `jacocoTestReport`, `jacocoTestCoverageVerification` complete without skipping. | Ensures compile + unit suites run holistically, satisfying AC2. | PERF-001 |

### AC3 – Execute full `integrationTest` suite (Testcontainers)

| ID | Level | Priority | Test | Justification | Mitigates |
|----|-------|----------|------|---------------|-----------|
| 1.4-INT-003 | Integration | P0 | Execute `test` job on both `ubuntu-latest` and `macos-latest`; confirm `integrationTest` uses Testcontainers and passes health checks. | Validates multi-platform container support and mitigates runner capability risk. | OPS-001 |

### AC4 – PR blocked if any gate fails

| ID | Level | Priority | Test | Justification | Mitigates |
|----|-------|----------|------|---------------|-----------|
| 1.4-E2E-002 | E2E | P0 | Configure branch protection requiring all CI jobs, intentionally break lint job, ensure PR cannot merge and status checks display failure. | Proves enforcement of merge blocking per AC4. | OPS-002 |

### Operational / Performance Observability

| ID | Level | Priority | Test | Justification | Mitigates |
|----|-------|----------|------|---------------|-----------|
| 1.4-INT-004 | Integration | P1 | Capture workflow duration metrics across 3 consecutive runs; ensure quality gates <15 min and mutation-testing <20 min, raising alert if thresholds exceeded. | Provides baseline to monitor PERF-001 over time. | PERF-001 |

## Risk Coverage Summary
- OPS-001 covered by 1.4-E2E-001 and 1.4-INT-003.
- OPS-002 covered by 1.4-E2E-002.
- TECH-001 addressed via 1.4-UNIT-001.
- PERF-001 addressed via 1.4-INT-002 and 1.4-INT-004.
- SEC-001 addressed via 1.4-INT-001.
