## Executive Summary

This test plan provides comprehensive validation procedures for Story 8.8 fix. The fix addresses a **P0 critical defect** where fuzz corpus persistence was completely non-functional, rendering incremental fuzzing impossible.

**Validation Scope:**
- Local fuzzing with corpus generation
- GitHub Actions corpus caching mechanics
- Incremental fuzzing across multiple runs
- Corpus growth tracking and monitoring

**Test Execution Time:** ~2 hours (including 30-minute nightly run wait)

### Test 1.4: Incremental Local Fuzzing
**Objective**: Verify corpus persists and grows across local runs

**AC Coverage**: AC5 (Incremental Fuzzing Works Across Runs), AC6 (Local Execution Works Identically)

**Preconditions**:
- Test 1.3 passed
- Existing corpus from previous run

**Test Steps**:
```bash
# Step 1: Record initial corpus count
INITIAL_COUNT=$(find framework/security/.jazzer/corpus -type f | wc -l)
echo "Initial corpus count: $INITIAL_COUNT"

# Step 2: Run second fuzzing session
./gradlew fuzzTest -Djazzer.flags="-max_total_time=60"

# Step 3: Record final corpus count
FINAL_COUNT=$(find framework/security/.jazzer/corpus -type f | wc -l)
echo "Final corpus count: $FINAL_COUNT"

# Step 4: Calculate growth
GROWTH=$((FINAL_COUNT - INITIAL_COUNT))
echo "Corpus growth: +$GROWTH files"
```

**Expected Results**:
- [ ] Final count > Initial count (corpus grew)
- [ ] Growth: +10 to +100 files (varies by complexity)
- [ ] No corpus reset (files not deleted)
- [ ] Execution time still ~60-70 seconds

**Pass Criteria**: Corpus growth ≥ 10 files

**Test Duration**: ~70 seconds

## Appendix B: Troubleshooting

### Issue: Corpus directory not created
**Symptom**: `ls: cannot access '.jazzer': No such file or directory`
**Solution**: Verify doFirst block in build.gradle.kts, check logs for creation message

### Issue: Execution time still <30 seconds
**Symptom**: Fuzz tests complete instantly
**Solution**: Verify jazzer.flags parsing, check JAZZER_FUZZ environment variable

### Issue: Cache save fails in CI
**Symptom**: "Path Validation Error" warning
**Solution**: Check corpus directory exists, verify GitHub Actions workflow paths

### Issue: Corpus not restored in Run #2
**Symptom**: "Cache not found" message
**Solution**: Verify cache key format, check SHA in cache key matches commit
