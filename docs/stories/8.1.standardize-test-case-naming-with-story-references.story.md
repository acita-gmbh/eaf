# Story 8.1: Standardize Test Case Naming with Story References

## Status
Approved

## Story
**As a** QA Engineer,
**I want** all test cases to follow a consistent naming convention with story references,
**so that** test coverage can be traced back to story acceptance criteria and quality gates can be properly validated.

## Business Context
During comprehensive Epic 8 investigation, inconsistent test case naming was identified as a quality and traceability gap:

**Current State:**
- **27% compliance rate** - Only 12 of 44 test files follow the story reference pattern
- **32 violations found** across tools, products, framework, and shared modules
- Mixed approaches: Some tests use `{STORY}-{TYPE}-{SEQ}: {Description}`, others have no story references

**Impact:**
- Difficult to trace test coverage back to story acceptance criteria
- Quality gate validation requires manual inspection
- Test reports don't clearly show story completion status
- New developers unsure of test naming conventions

**Examples of Current Inconsistency:**
- WidgetQueryHandlerTest: ✅ `test("2.4-UNIT-001: FindWidgetByIdQuery returns widget response when found")`
- WidgetTest (BehaviorSpec): ❌ No story references in Given/When/Then blocks
- TenantEventMessageInterceptorSpec: ⚠️ MIXED - 3 tests with IDs, 3 without

## Acceptance Criteria

**Phase 1: Define Standard**
1. Test naming standard documented with clear patterns:
   - FunSpec: `test("{STORY}-{TYPE}-{SEQ}: {Description}")`
   - BehaviorSpec: Comment header `// {STORY}-{TYPE}-{SEQ}` above Given/When/Then blocks
   - Pattern format: `{EPIC}.{STORY}-{TYPE}-{SEQ}` (e.g., "2.4-UNIT-001", "4.6-INT-003")
   - TYPE values: UNIT, INT (integration), E2E (end-to-end)
2. Examples provided for each test framework in documentation
3. Standard added to test-strategy-and-standards-revision-3.md

**Phase 2: Konsist Enforcement**
4. Konsist rule created: `TestCaseNamingConventionTest.kt`
5. Rule validates FunSpec test names match pattern: `^\d+\.\d+-[A-Z]+-\d+:.*`
6. Rule validates BehaviorSpec blocks have story reference comments
7. Rule integrated into `konsistTest` task
8. CI pipeline fails if naming violations detected

**Phase 3: Codebase Migration**
9. All FunSpec tests updated with story references (32 files)
10. All BehaviorSpec tests updated with story reference comments
11. TenantEventMessageInterceptorSpec fully consistent (3 missing IDs added)
12. Zero naming violations in `./gradlew jvmKotest`
13. **100% compliance rate** achieved (all test files follow standard)

**Phase 4: Process Integration**
14. Updated test templates in eaf-cli to include story references
15. Developer documentation updated with naming examples
16. Pre-commit hook (optional) warns on missing story references

### Test Naming Standard (Reference)

**Mandatory Pattern**: `{EPIC}.{STORY}-{TYPE}-{SEQ}: {Description}`

**FunSpec Example:**
```kotlin
test("2.4-UNIT-001: FindWidgetByIdQuery returns widget response when found") {
    // Test logic validates Story 2.4 AC1
}
```

**BehaviorSpec Example:**
```kotlin
// Story 2.1-UNIT-001: Widget aggregate creation with valid data
Given("Widget aggregate creation") {
    When("creating a widget with valid data") {
        Then("widget should be created successfully") {
            // Test logic
        }
    }
}
```
