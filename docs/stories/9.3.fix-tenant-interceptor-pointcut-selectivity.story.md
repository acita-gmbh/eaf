# Story 9.3: Fix TenantDatabaseSessionInterceptor Pointcut Selectivity

## Status
Draft

## Story
**As a** Developer,
**I want** the TenantDatabaseSessionInterceptor to only intercept application @Transactional methods,
**so that** Spring infrastructure methods (Modulith, JPA) can execute without requiring tenant context.

## Business Context

**Discovered During**: Story 9.2 browser validation (AC9 testing - POST /widgets)

**Impact**:
- ❌ POST /widgets returns HTTP 400 (widget creation fails)
- ❌ Backend startup shows warnings during Spring Modulith cleanup
- ❌ Spring infrastructure methods crash when no HTTP request context available

**Root Cause**:

The current interceptor pointcut is overly broad:
```kotlin
@Around("@annotation(org.springframework.transaction.annotation.Transactional)")
```

This matches **ALL** @Transactional methods including:
- ✅ Application code: `WidgetQueryHandler`, `WidgetProjectionRepository` (needs tenant context)
- ❌ Spring infrastructure: `JpaEventPublicationRepository`, Spring Modulith beans (no tenant context available)

**Error Evidence**:
```
java.lang.IllegalStateException: Missing or invalid tenant_id claim in JWT token
at TenantDatabaseSessionInterceptor.setSessionVariableBeforeTransaction
at JpaEventPublicationRepository$$SpringCGLIB$$0.findIncompletePublications
at DefaultEventPublicationRegistry.destroy
```

**Current Behavior**:
- Interceptor calls `tenantContext.getCurrentTenantId()` (fail-closed)
- Spring infrastructure methods have no HTTP request → no JWT → no tenant context
- `getCurrentTenantId()` throws IllegalStateException
- Application startup/shutdown fails

**Desired Behavior**:
- Interceptor only affects `com.axians.eaf` application packages
- Spring infrastructure methods execute normally without tenant context
- Multi-tenant isolation still enforced for application code

## Acceptance Criteria

1. TenantDatabaseSessionInterceptor uses package-selective pointcut
2. Pointcut only matches @Transactional methods in `com.axians.eaf.*` packages
3. Spring Modulith methods (`org.springframework.modulith.*`) are NOT intercepted
4. Spring Data JPA methods (`org.springframework.data.jpa.*`) are NOT intercepted
5. Application startup completes without tenant context errors
6. Application shutdown completes without tenant context errors
7. POST /widgets successfully creates widget (AC9 from Story 9.2)
8. Created widget appears in GET /widgets list (AC9 from Story 9.2)
9. WidgetQueryHandler still has tenant context set (regression test)
10. Integration tests pass with new pointcut

## Dev Notes

### Recommended Pointcut Patterns

**Option 1: Package-Based Execution** (Recommended):
```kotlin
@Around("execution(* com.axians.eaf..*.*(..))" +
    " && @annotation(org.springframework.transaction.annotation.Transactional)")
```
- Matches: All methods in `com.axians.eaf` packages with @Transactional
- Excludes: Spring framework packages (`org.springframework.*`)

**Option 2: Within Package** (Alternative):
```kotlin
@Around("within(com.axians.eaf..*)" +
    " && @annotation(org.springframework.transaction.annotation.Transactional)")
```
- Matches: @Transactional methods where declaring class is in `com.axians.eaf`
- Simpler expression, same effect

**Option 3: Exclude Known Infrastructure** (Most Defensive):
```kotlin
@Around("@annotation(org.springframework.transaction.annotation.Transactional)" +
    " && !within(org.springframework.modulith..*)" +
    " && !within(org.springframework.data.jpa..*)")
```
- Explicitly excludes known problematic packages
- More brittle (need to add exclusions as discovered)

### Testing Strategy

**Unit Test** (with Nullable Pattern):
```kotlin
test("pointcut should only match application packages") {
    val interceptor = TenantDatabaseSessionInterceptor(...)

    // Should match: com.axians.eaf.products.widgetdemo.query.WidgetQueryHandler
    // Should NOT match: org.springframework.modulith.events.jpa.JpaEventPublicationRepository
}
```

**Integration Test** (with Testcontainers):
```kotlin

    val request = CreateWidgetRequest(name = "Test", value = 100, category = "test")

```
