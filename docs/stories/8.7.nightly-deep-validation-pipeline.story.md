## Story
**As a** Developer,
**I want** nightly deep validation pipeline separated from PR feedback pipeline,
**so that** I get fast PR feedback (<3 minutes) while maintaining comprehensive quality validation through nightly scheduled deep testing.

---

## Business Context

### Problem

Story 8.6 established multi-stage testing infrastructure but currently runs ALL testing (including mutation testing) on every PR. This means:

- **PR feedback time**: 6-8 minutes (mutation testing takes 3-5 min)
- **Developer friction**: Waiting for mutation analysis on every commit
- **CI resource waste**: Running deep validation 50+ times per day on every commit
- **Delayed feedback**: Integration tests + mutation testing serialized in PR pipeline

**Industry Standard** (Google, Netflix, Microsoft):
- **Fast feedback** (<3 min): Unit + Integration tests only
- **Deep validation** (nightly): Mutation, Property-based, Fuzz testing
- **Separation rationale**: Different cadences for different validation goals

### Opportunity

Move mutation testing, property testing, and fuzz testing to a **scheduled nightly pipeline** to:

1. **Accelerate PR feedback**: 6-8 min → <3 min (57% improvement)
2. **Optimize CI resources**: Deep validation once per day vs 50+ times
3. **Enable longer-running tests**: Fuzz testing can run 30+ minutes nightly
4. **Maintain quality**: Regressions still caught within 24 hours
5. **Improve developer experience**: Faster iteration cycles

```yaml
jobs:
  quality-gates:

      - run: ./gradlew ciTest ciIntegrationTest koverVerify  # 6-8 min
```

**Execution Time**: 6-8 minutes
**Runs**: 50+ times per day across all PRs
**Issue**: Mutation testing (3-5 min) blocks fast feedback

### Nightly Pipeline (Not Implemented)
- ❌ Property-based tests not running
- ❌ Fuzz tests not running
- ❌ No scheduled deep validation

## Target State (Story 8.7)

### PR Pipeline (Fast Feedback - <3 minutes)
```yaml
jobs:
  fast-feedback:
    steps:
      - run: ./gradlew ktlintCheck detekt konsistTest  # Quality gates
      - run: ./gradlew test integrationTest            # Unit + Integration
      - run: ./gradlew koverXmlReport koverVerify      # Coverage
```

**Execution Time**: 2-3 minutes ✅ **57% improvement**
**Runs**: Every commit
**Focus**: Code quality + functional correctness

---

### Nightly Pipeline (Deep Validation - 15-30 minutes)
```yaml
jobs:
  nightly-deep-validation:
    schedule:
      - cron: '0 2 * * *'  # 2 AM UTC daily
    steps:
      - run: ./gradlew test integrationTest            # Baseline
      - run: ./gradlew propertyTest                    # Property-based
      - run: ./gradlew fuzzTest --args="-max_total_time=1800"  # 30 min fuzz
      - run: ./gradlew pitest                          # Mutation testing
      - run: ./gradlew koverXmlReport                  # Coverage
```

**Execution Time**: 15-30 minutes (acceptable for nightly)
**Runs**: Once per day at 2 AM UTC
**Focus**: Deep validation, edge cases, security

## Implementation Notes

### Workflow Schedule Strategy

**Recommended Schedule**:
```yaml
schedule:
  - cron: '0 2 * * *'  # 2 AM UTC = Off-peak hours
```

**Rationale**:
- Minimal CI resource contention
- Results available by morning standup (EU timezone)
- Doesn't interfere with active development hours

**Alternative Schedules**:
- `0 2 * * 1-5`: Weekdays only (skip weekends)
- `0 2,14 * * *`: Twice daily (morning + afternoon validation)

### Fuzz Test Corpus Management

**Strategy**:
```yaml
- name: Restore fuzz corpus
  uses: actions/cache/restore@v4
  with:
    path: |
      framework/security/.jazzer/corpus
    key: fuzz-corpus-${{ github.sha }}
    restore-keys: |
      fuzz-corpus-

- name: Run fuzz tests
  run: ./gradlew fuzzTest --args="-max_total_time=1800"

- name: Save fuzz corpus
  uses: actions/cache/save@v4
  with:
    path: |
      framework/security/.jazzer/corpus
      framework/security/.jazzer/crashes
    key: fuzz-corpus-${{ github.sha }}
```

**Benefits**:
- Incremental fuzzing (builds on previous runs)
- Crash reports preserved
- Coverage improves over time
