# Story 7.3: Create "New Aggregate" Generator

## Status
Done

## Story
**As a** Developer,
**I want** the CLI to have a `scaffold aggregate` command,
**so that** I can generate a complete CQRS/ES vertical slice (command, event, aggregate, projection, and API stubs) in seconds, adhering to all framework patterns.

## Acceptance Criteria
1. Running `eaf scaffold aggregate <Name> --module <moduleName>` generates all required Kotlin files for a new Axon Aggregate inside the target module.
2. The generated files MUST include templates/stubs for:
   - The Domain Aggregate (`<Name>Aggregate.kt`) with Axon annotations.
   - The API (Commands, Events, and Queries files for the aggregate).
   - The Projection/EventHandler (`<Name>Projection.kt`).
   - API Endpoints (stubs for the Spring Controllers and Query Handlers).
3. The generated templates must correctly include all required patterns (e.g., Tenancy checks from Epic 4, Logging from Epic 6).
4. The generated code (including generated test stubs) must pass all Constitutional TDD and Quality Gates (from Story 1.2) immediately upon generation.

### Aggregate Implementation Pattern

**Domain Aggregate** (`Widget.kt` as template reference):

```kotlin
@Aggregate
class Widget {
    @AggregateIdentifier
    private lateinit var widgetId: String
    private lateinit var tenantId: String
    // Business fields...

    constructor() // Required by Axon

    @CommandHandler
    constructor(command: CreateWidgetCommand) {
        val currentTenant = TenantContext().getCurrentTenantId()
        require(command.tenantId == currentTenant) {
            "Access denied: tenant context mismatch"
        }
        validateCreateCommand(command).fold(
            { error -> throw IllegalArgumentException("Validation failed: $error") },
            {
                AggregateLifecycle.apply(
                    WidgetCreatedEvent(/* fields */)
                )
            }
        )
    }

    @CommandHandler
    fun handle(command: UpdateWidgetCommand): Either<WidgetError, Unit> {
        val currentTenant = TenantContext().getCurrentTenantId()
        when {
            command.tenantId != currentTenant -> return WidgetError.TenantIsolationViolation(...).left()
            this.tenantId != currentTenant -> return WidgetError.TenantIsolationViolation(...).left()
            // Business rule checks...
        }
        return validateUpdateCommand(command).fold(
            { error -> error.left() },
            {
                AggregateLifecycle.apply(WidgetUpdatedEvent(/* fields */))
                Unit.right()
            }
        )
    }

    @EventSourcingHandler
    fun on(event: WidgetCreatedEvent) {
        this.widgetId = event.widgetId
        this.tenantId = event.tenantId
        // Set all fields...
    }

    @EventSourcingHandler
    fun on(event: WidgetUpdatedEvent) {
        event.name?.let { this.name = it }
        // Update nullable fields...
        this.updatedAt = event.updatedAt
    }

    companion object {
        fun validateCreateCommand(command: CreateWidgetCommand): Either<WidgetError, Unit> {
            // Validation logic with regex patterns, range checks
        }
    }
}
```

**Critical Pattern Elements**:
1. ✅ Empty constructor for Axon framework
2. ✅ @CommandHandler constructor for creation (throws on validation failure)
3. ✅ @CommandHandler methods for updates (return Arrow Either)
4. ✅ Tenant validation in ALL command handlers (fail-closed)
5. ✅ @EventSourcingHandler methods for state reconstitution
6. ✅ Companion object with validation logic
7. ✅ Arrow Either for error handling (domain returns Either.Left, handler converts)

### Projection Entity Pattern

```kotlin
@Entity
@Table(
    name = "widget_projection",
    indexes = [
        Index(name = "idx_widget_projection_tenant_id", columnList = "tenant_id"),
        Index(name = "idx_widget_projection_category", columnList = "category"),
        Index(name = "idx_widget_projection_created_at", columnList = "created_at"),
    ],
)
data class WidgetProjection(
    @Id
    @Column(name = "widget_id", nullable = false, length = 36)
    val widgetId: String,

    @Column(name = "tenant_id", nullable = false, length = 36)
    private val tenantId: String,

    @Column(name = "name", nullable = false, length = 255)
    val name: String,

    @Column(name = "value", nullable = false, precision = 19, scale = 2)
    val value: BigDecimal,

    @Column(name = "created_at", nullable = false)
    val createdAt: Instant,

    @Column(name = "updated_at", nullable = false)
    val updatedAt: Instant,
) : TenantAware {
    constructor() : this(/* defaults */)  // JPA requirement

    override fun getTenantId(): String = tenantId

    fun update(
        name: String = this.name,
        value: BigDecimal = this.value,
    ): WidgetProjection = copy(
        name = name,
        value = value,
        updatedAt = Instant.now(),
    )
}
```

**Critical Elements**:
1. ✅ Implements TenantAware interface
2. ✅ Indexes on tenant_id (required for multi-tenancy)
3. ✅ Private tenantId field (no public setter)
4. ✅ Empty constructor for JPA
5. ✅ update() method returns new instance (immutability)

### Projection Handler Pattern

```kotlin
@Component
@ProcessingGroup("widget-projection")
class WidgetProjectionHandler(
    private val repository: WidgetProjectionRepository,
    private val objectMapper: ObjectMapper,
) {
    private val logger = LoggerFactory.getLogger(WidgetProjectionHandler::class.java)

    @EventHandler
    @Transactional
    fun on(event: WidgetCreatedEvent) {
        logger.debug(
            "Processing WidgetCreatedEvent for widgetId: {} in tenant: {}",
            event.widgetId,
            event.tenantId,
        )

        try {
            val projection = WidgetProjection(/* map event fields */)
            repository.save(projection)

            logger.debug("Successfully created widget projection for widgetId: {}", event.widgetId)
        } catch (exception: PersistenceException) {
            logger.error("Failed to process WidgetCreatedEvent: {}", exception.message, exception)
            throw exception  // Re-throw for Axon retry
        }
    }
```

### Custom Validation Pattern Support (Enhancement)

**CLI Validation Format**: `--validation "sku:^[A-Z]{3}-[0-9]{6}$, quantity:1:1000, status:ACTIVE|INACTIVE|ARCHIVED"`

**Pattern Types**:
- **Regex Pattern** (String fields): `fieldName:^regex$`
  - Example: `sku:^[A-Z]{3}-[0-9]{6}$` generates `SKU_PATTERN = Regex("^[A-Z]{3}-[0-9]{6}$")`
- **Range Validation** (Numeric fields): `fieldName:min:max`
  - Example: `quantity:1:1000` generates `MIN_QUANTITY = 1`, `MAX_QUANTITY = 1000`
- **Enum Validation** (String fields): `fieldName:value1|value2|value3`
  - Example: `status:ACTIVE|INACTIVE` generates `VALID_STATUSES = setOf("ACTIVE", "INACTIVE")`

**Generated Validation Code**:
```kotlin
companion object {
    private val SKU_PATTERN = Regex("^[A-Z]{3}-[0-9]{6}$")
    private const val MIN_QUANTITY = 1
    private const val MAX_QUANTITY = 1000
    private val VALID_STATUSES = setOf("ACTIVE", "INACTIVE", "ARCHIVED")

    private fun validateSku(sku: String): DomainError.ValidationError? =
        if (!sku.matches(SKU_PATTERN)) {
            DomainError.ValidationError(
                field = "sku",
                constraint = "pattern",
                invalidValue = sku,
            )
        } else null

    private fun validateQuantity(quantity: Int): DomainError.ValidationError? =
        if (quantity < MIN_QUANTITY || quantity > MAX_QUANTITY) {
            DomainError.ValidationError(
                field = "quantity",
                constraint = "range",
                invalidValue = quantity,
            )
        } else null

    private fun validateStatus(status: String): DomainError.ValidationError? =
        if (!VALID_STATUSES.contains(status)) {
            DomainError.ValidationError(
                field = "status",
                constraint = "enum",
                invalidValue = status,
            )
        } else null
}
```

**Default Validation** (when --validation not specified):
- String fields: `^[A-Za-z0-9](?:[A-Za-z0-9 _-]{0,98}[A-Za-z0-9])?$` (2-100 chars)
- Numeric fields: `>= 0` (non-negative)
- No enum validation by default
