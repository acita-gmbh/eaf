## Blocking Issues

### Architectural Incompatibility

**Primary Blocker**: Kotest 6.0.3 FunSpec + @SpringBootTest + Custom integrationTest Source Set = INCOMPATIBLE

**Technical Details**:
- Kotest Gradle plugin (io.kotest v6.0.3) does NOT register Kotest DSL for custom source sets created by convention plugins
- TestingConventionPlugin creates `integrationTest` source set but doesn't configure Kotest for it
- Result: Kotlin compiler cannot resolve Kotest DSL functions (`test`, `context`, `beforeEach`) in integrationTest source set
- Symptom: 150+ "Unresolved reference 'test'" compilation errors on all @SpringBootTest integration tests

**Why Tests Were Actually Disabled**:
- Tests were disabled in Story 3.4 (commit 612acfd), NOT Story 4.5
- Reason: Product-level E2E tests with @SpringBootTest don't belong in framework modules
- Tests were already broken/incompatible when migrated in Story 4.5
- Story 4.6 incorrectly assumed "just update application reference"

### Required Prerequisite Work

**Before Story 4.6 can proceed**, one of the following must be completed:

**Option A**: Fix Kotest + Spring Boot Integration (Recommended)
- New Story: "Configure Kotest Gradle Plugin for integrationTest Source Set"
- Modify TestingConventionPlugin to call `configure<KotestPluginExtension> { sourceSets.add(integrationTest) }`
- Requires: Add KotestPluginExtension to convention plugin classpath
- Impact: Framework-wide change affecting all modules
- Estimated Effort: 4-6 hours
- Risk: Medium (could break existing test infrastructure)

**Option B**: Convert Tests to JUnit 5
- Rewrite all @SpringBootTest tests using JUnit 5 @Test annotations
- Violates CLAUDE.md mandate: "Use Kotest, NEVER JUnit"
- Estimated Effort: 8-12 hours
- Risk: Low (JUnit 5 + Spring Boot is well-established pattern)

**Option C**: Simplify to Axon Fixtures Only (Implemented but reverted)
- Rewrite all tests using AggregateTestFixture (no Spring Boot context)
- Loses: REST API testing, real database validation, E2E CQRS flow
- Estimated Effort: 6-8 hours
- Risk: Low (working pattern exists)
- Trade-off: Reduced test coverage (no HTTP endpoints, no projections, no real infrastructure)

### Recommendation

**Mark Story 4.6 as BLOCKED** until prerequisite work is completed.

Create **new Story 4.6-A**: "Enable Kotest DSL for integrationTest Source Set"
- Prerequisites: None
- Estimated Effort: 4-6 hours
- Scope: Modify TestingConventionPlugin.kt to configure Kotest for integrationTest source set
- Impact: Framework infrastructure (all modules benefit)

Then retry **Story 4.6** (re-enable widget integration tests) after 4.6-A completes.
