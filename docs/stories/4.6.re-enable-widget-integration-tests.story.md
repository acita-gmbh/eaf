# Story 4.6: Re-enable Widget Integration Tests in widget-demo

## Status
Ready for Implementation

## Story
**As a** Developer,
**I want** the 5 disabled Widget integration tests re-enabled in products/widget-demo,
**so that** we have comprehensive E2E validation of the Widget domain in a product context with full CQRS flow coverage.

## Business Context
During Story 4.5 migration, 5 integration tests were successfully migrated from framework/widget to products/widget-demo but remained disabled because they:
1. Referenced LicensingServerApplication (Epic 8 - doesn't exist yet)
2. Require full infrastructure (PostgreSQL, Keycloak, Redis Testcontainers)
3. Were originally designed for product-level E2E validation (not framework-level)

Now that Widget domain lives in products/widget-demo, these tests CAN be enabled by updating them to use WidgetDemoApplication and configuring the necessary test infrastructure.

4. All 5 integration test suites compile without errors

6. Integration tests added to CI pipeline (or documented how to run locally)
7. Test execution time documented and within acceptable range (<5 minutes total)
8. No regressions in existing active integration tests (TenantBoundaryValidationIntegrationTest)

### Test File Details

**WidgetIntegrationTest.kt** (193 lines):
- Tests aggregate lifecycle (create, update, delete)
- Tests tenant isolation violations
- Tests concurrent operations
- Tests business rule validation
- Tests partial updates
- Uses programmatic Axon configuration (DefaultConfigurer)

**WidgetApiIntegrationTest.kt** (245 lines):
- REST API integration tests
- POST /widgets - create widget
- GET /widgets/{id} - retrieve widget
- PUT /widgets/{id} - update widget  
- DELETE /widgets/{id} - delete widget
- Uses MockMvc for HTTP testing
- Requires full Spring Boot context

**WidgetWalkingSkeletonIntegrationTest.kt** (305 lines):
- Epic 2 completion validation

- Performance target: <500ms end-to-end
- Validates eventual consistency
- Uses MockMvc + real database

**persistence/WidgetEventStoreIntegrationTest.kt** (278 lines):
- Event store persistence validation

- Snapshot testing
- PostgreSQL event store validation

**projections/WidgetEventProcessingIntegrationTest.kt**:
