## Story

### Testing Standards

**Test Framework**: Kotest 6.0.3 (FunSpec style) - **MANDATORY**

**Test Location**: `products/widget-demo/src/integration-test/kotlin/`

**Test Execution**:
- Integration tests run via `./gradlew :products:widget-demo:integrationTest`
- Uses JUnit Platform for custom source sets (per Kotest 6.0.3 migration pattern)
- Native Kotest runner not applicable to integration-test source set

**TestContainers Setup**:
- PostgreSQL, Keycloak, Redis from `shared/testing` module
- Kotest lifecycle: `listener(TestContainers.postgres.perSpec())`
- Containers start once per test spec (shared across tests in same class)

**Critical Anti-Patterns**:
- ❌ **NEVER mix JUnit and Kotest annotations** - JUnit `@Disabled` has no effect on Kotest tests
- ❌ **NO mocking of domain logic** - Only infrastructure adapters
- ❌ **NO H2 database** - PostgreSQL Testcontainers only for RLS validation

**Performance Target**: <5 minutes total execution time for all integration tests

**Test Profile**: `@ActiveProfiles("test")` for test-specific configuration

**Spring Context**: Full `@SpringBootTest` with `WidgetDemoApplication::class` (only where explicit reference needed)
