# Story 1.1: Initialize Gradle Monorepo Structure

## Status
Draft

## Story
**As a** Core Developer,
**I want** a defined Gradle Multi-Module Monorepo structure configured with version catalogs,
**so that** all framework libraries, product apps (like React-Admin), and shared code are managed consistently in one place.

## Acceptance Criteria
1. The project root is initialized with a Gradle wrapper and a `build-logic` directory for convention plugins.
2. The monorepo includes the required directory structure (`framework/`, `products/`, `shared/`, `apps/admin/`) defined in the architecture.
3. A central Gradle version catalog (`gradle/libs.versions.toml`) is created and populated with the exact, locked stack versions (Kotlin 2.0.10, Spring Boot 3.3.5, Axon 4.9.4, Kotest).
4. All shared build logic (including dependency versions and plugin configurations) is defined via convention plugins within `build-logic` to ensure consistency.

### Version Catalog Requirements

- Mandatory entries include Kotlin 2.0.10, JVM 21, Spring Boot 3.3.5, Spring Modulith 1.3.0, Axon 4.9.4, Arrow 1.2.4, PostgreSQL 42.7.4, jOOQ 3.19.15, Kotest 5.9.1, Testcontainers 1.20.4, ktlint 1.4.2, Detekt 1.23.7, Konsist 0.18.0, Pitest 1.15.0. `[Source: architecture/tech-stack.md#critical-version-constraints]`

### Convention Plugin Requirements
1. **Kotlin Convention Plugin** – Applies Kotlin JVM plugin, enforces JVM 21, strict compiler options, ktlint, detekt. `[Source: architecture/coding-standards-revision-2.md#kotlin-standards]`
2. **Spring Boot Convention Plugin** – Applies Spring Boot + Modulith plugins, configures Actuator, validation, security defaults. `[Source: architecture/high-level-architecture.md#system-architecture]`
3. **Testing Convention Plugin** – Registers Kotest, Nullable pattern helpers, Testcontainers integration tests. `[Source: architecture/test-strategy-and-standards-revision-3.md#testing-framework-standards]`
4. **Quality Gates Plugin** – Wires ktlintCheck, detekt, konsistTest, pitest as zero-tolerance gates. `[Source: architecture/tech-stack.md#quality-gate-configuration]`

### Testing Standards
- Constitutional TDD mandates test-first development, Nullable pattern for business logic, and Testcontainers for integrations. `[Source: architecture/test-strategy-and-standards-revision-3.md#testing-philosophy]`
- Kotest is mandatory; JUnit annotations are forbidden. `[Source: architecture/test-strategy-and-standards-revision-3.md#testing-framework-standards]`
- Coverage thresholds: ≥85% line and ≥80% mutation coverage enforced via Jacoco + Pitest. `[Source: architecture/tech-stack.md#quality-gate-configuration]`
- CI pipeline must run ktlint, Detekt, Konsist, unit, integration, and Pitest suites. `[Source: architecture/development-workflow.md#continuous-integration-integration]`

### Technical Constraints
- Zero tolerance for wildcard imports, generic exceptions, or hard-coded dependency versions outside the catalog. `[Source: architecture/coding-standards-revision-2.md#critical-requirements-must-follow]`
- JVM toolchain must be 21 LTS across all modules; Gradle convention plugins enforce this. `[Source: architecture/tech-stack.md#critical-version-constraints]`
- Convention plugins are the only sanctioned path to propagate shared build logic. `[Source: architecture/tech-stack.md#build-system]`
