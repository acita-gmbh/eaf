## Dev Notes

### Story 4.6 Reference

**Solution Pattern Discovered in Story 4.6:**
- **File**: `framework/security/src/integration-test/kotlin/com/axians/eaf/framework/security/filters/TenantContextFilterIntegrationTest.kt`
- **Pattern**: @Autowired field injection + init block (NOT constructor injection)
- **Critical**: Plugin order matters - `id("eaf.testing")` BEFORE `id("eaf.spring-boot")`

**Anti-Pattern to Avoid:**
```kotlin
// ❌ WRONG - Causes circular dependency
@SpringBootTest
class MyTest(
    private val mockMvc: MockMvc  // Constructor injection fails
) : FunSpec({ ... })
```

**Correct Pattern:**
```kotlin
// ✅ CORRECT - Field injection works
@SpringBootTest
@ActiveProfiles("test")
class MyTest : FunSpec() {
    @Autowired
    private lateinit var mockMvc: MockMvc

    init {
        extension(SpringExtension())
        test("my test") { /* mockMvc available */ }
    }
}
```

### Testcontainers Configuration Pattern

**Standard Pattern** (from working tests):
```kotlin
@SpringBootTest
@ActiveProfiles("test")
class MyIntegrationTest : FunSpec() {
    @Autowired
    private lateinit var dependency: SomeDependency

    init {
        extension(SpringExtension())

        test("8.3-INT-001: Test description") {
            // Test logic using autowired dependencies
        }
    }

    companion object {
        @Container
        private val postgres = PostgreSQLContainer<Nothing>("postgres:16.1")
            .withDatabaseName("eaf_test")
            .withUsername("eaf")
            .withPassword("eaf")

        @Container
        private val keycloak = KeycloakContainer("quay.io/keycloak/keycloak:25.0.6")
            .withRealmImportFile("keycloak-realm-eaf-test.json")

        @JvmStatic
        @DynamicPropertySource
        fun configureProperties(registry: DynamicPropertyRegistry) {
            postgres.start()
            keycloak.start()

            registry.add("spring.datasource.url") { postgres.jdbcUrl }
            registry.add("spring.datasource.username") { postgres.username }
            registry.add("spring.datasource.password") { postgres.password }
            registry.add("spring.security.oauth2.resourceserver.jwt.issuer-uri") {
                "${keycloak.authServerUrl}/realms/eaf"
            }
        }
    }
}
```

### Plugin Order Critical Configuration

**products/widget-demo/build.gradle.kts:**
```kotlin
plugins {
    id("eaf.testing")      // FIRST - Kotest setup
    id("eaf.spring-boot")  // SECOND - Spring Boot
}
```

**Rationale**: Kotest configuration must be applied before Spring Boot to avoid compilation errors (150+ errors if reversed).

### Expected Test Failures and Fixes

**Common Issues from Story 4.6:**
1. **Constructor injection**: Convert to @Autowired field injection
2. **Missing SpringExtension**: Add `extension(SpringExtension())` in init block
3. **Container lifecycle**: Ensure `start()` called in @DynamicPropertySource
4. **Plugin order**: Verify eaf.testing before eaf.spring-boot
5. **Application class**: Tests may reference wrong application class (LicensingServerApplication vs WidgetDemoApplication)

### Investigation Strategy

**Per-Test Approach:**
1. Read disabled test file to understand original purpose
2. Identify why it was disabled (check git history, comments)
3. Apply Story 4.6 solution pattern
4. Run test locally to identify any remaining issues
5. Fix issues iteratively
6. Document test purpose and coverage
7. Move to kotlin/ directory

**Timebox**: 1 day maximum per test
- If test unfixable within 1 day, document issue and escalate
- Keep test disabled if fundamental architecture change needed
