# Story 5.1: Implement Standardized Structured Logging

## Status
Approved

## Story
**As an** Operator,
**I want** all framework and application logs to be output in a standardized, structured JSON format,
**so that** they can be easily ingested and parsed by an external logging stack (like Loki/Grafana, per the prototype roadmap).

## Acceptance Criteria
1. A logging convention (e.g., via a Logback/Logstash encoder configuration) is defined in the `build-logic` convention plugins (from Epic 1) and applied to all modules.
2. All log output MUST be in a structured JSON format.
3. All log entries MUST automatically include critical context fields: `service_name`, `trace_id`, and `tenant_id` (retrieved from the `TenantContext` [from Story 4.1]).
4. Integration tests confirm that logs are written in the correct JSON format and include the required context (especially the propagated `tenant_id`).

## Tasks / Subtasks

- [ ] Task 1: Create Logback JSON configuration in build-logic convention plugins (AC: 1)
  - [ ] Create logback-eaf.xml configuration file with JSON encoder
  - [ ] Configure Logback/Logstash encoder for structured JSON output
  - [ ] Add logback configuration to build-logic convention plugins
  - [ ] Apply logging convention to all modules via gradle plugins

- [ ] Task 2: Implement context field injection (AC: 3)
  - [ ] Create LoggingContextProvider to extract service_name, trace_id, tenant_id

  - [ ] Configure JSON encoder with required field structure
  - [ ] Test log output format across different log levels

  - [ ] Test structured JSON format output

### Data Models
**Logging Context Fields** [Source: epic-5-observability-core-collection.md#story-5.1]:
- `service_name`: Application identifier for multi-service environments
- `trace_id`: Distributed tracing correlation ID
- `tenant_id`: Multi-tenant isolation context from TenantContext (Story 4.1)

**Log Entry Structure**:
```json
{
  "timestamp": "2025-09-29T10:15:30.123Z",
  "level": "INFO",
  "logger": "com.axians.eaf.products.widgetdemo.WidgetController",
  "message": "Widget created successfully",
  "service_name": "widget-demo",
  "trace_id": "abc123def456",
  "tenant_id": "tenant-a",
  "thread": "http-nio-8080-exec-1"
}
```
