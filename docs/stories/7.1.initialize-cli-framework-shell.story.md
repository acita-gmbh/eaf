## Dev Notes

### Story Context

This is the first story in Epic 7 (Scaffolding CLI v1), which delivers the core Developer Experience promise of the EAF framework. The CLI will automate creation of all boilerplate required by the complex Hexagonal/CQRS/Modulith/Flowable stack, enabling new developers to meet the <1 month productivity goal. This story creates the foundational CLI shell that future stories will build upon to add specific generators (modules, aggregates, React-Admin resources).

### Previous Story Insights

Story 6.6 completed Epic 6 (Core Framework Hooks - Flowable Prep), establishing the Dockets Pattern BPMN template for workflow orchestration. The CLI developed in this epic will eventually generate BPMN templates, aggregate scaffolds, and other framework patterns established in Epics 1-6.[Source: docs/stories/6.6.implement-dockets-pattern-bpmn-template.story.md#completion]

### Technology Selection Rationale

**Picocli Framework (v4.7.6)**:
- **Type-Safe CLI**: Kotlin-friendly annotation-based framework with compile-time validation
- **Subcommand Support**: Natural fit for `eaf scaffold module|aggregate|ra-resource` command structure
- **Auto-Generated Help**: Built-in `--help` generation reduces documentation maintenance
- **Industry Standard**: Used by Gradle, Micronaut CLI, Spring Native - proven for code generation tools
[Source: Picocli documentation - code generation use case patterns]

**Mustache.java Templating (v0.9.14)**:
- **Logic-Less Templates**: Forces separation of template logic from generation code
- **Simple Syntax**: `{{variable}}` placeholders easy for developers to understand and maintain
- **Lightweight**: 100KB dependency with no transitive dependencies
- **Kotlin Compatibility**: Pure Java library with excellent Kotlin interoperability
- **Alternative Considered**: Velocity (rejected - heavier dependency, more complex syntax, security concerns with arbitrary code execution)
[Source: architecture/tech-stack.md - lightweight tooling preference]

### Templating Engine Integration Pattern

**Template Engine Wrapper**:
```kotlin
class TemplateEngine {
    private val mustacheFactory = DefaultMustacheFactory()

    fun render(templateName: String, context: Map<String, Any>): String {
        val mustache = mustacheFactory.compile("templates/$templateName")
        val writer = StringWriter()
        mustache.execute(writer, context)
        return writer.toString()
    }
}
```

**Test Template (Smoke Test)**:
```mustache
// templates/test-template.mustache
Hello, {{name}}!
This template validates Mustache integration.
Version: {{version}}
```

**Test Validation**:
```kotlin
class TemplateEngineTest : FunSpec({
    test("should render test template with context") {
        val engine = TemplateEngine()
        val context = mapOf(
            "name" to "EAF Developer",
            "version" to "0.1.0"
        )

        val result = engine.render("test-template.mustache", context)

        result shouldContain "Hello, EAF Developer!"
        result shouldContain "Version: 0.1.0"
    }
})
```
