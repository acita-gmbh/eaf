# Story 5.2: Implement Prometheus Metrics (Micrometer)

## Status
Done

## Story
**As an** Operator,
**I want** all services built on the EAF to expose standardized application metrics,
**so that** we can monitor system health via a Prometheus-compatible endpoint.

## Acceptance Criteria
1. The Spring Boot Actuator, Micrometer, and Prometheus registry dependencies are added and managed via the convention plugins.
2. The `/actuator/prometheus` endpoint is enabled, secured (requires AuthN from Epic 3), and exposed.
3. Core metrics (JVM, HTTP server requests, Axon message processing) are exposed by default.
4. All relevant metrics are automatically tagged with `tenant_id` (where applicable) and `service_name` to allow for granular filtering.

## Dev Notes

### Implementation Highlights
- `build-logic/src/main/kotlin/conventions/ObservabilityConventionPlugin.kt`: reusable plugin that attaches Micrometer core and Prometheus registry dependencies to any module opting in; keeps dependency versions centralized in `gradle/libs.versions.toml`.
- `framework/observability/src/main/kotlin/com/axians/eaf/framework/observability/metrics/MetricsConfiguration.kt`: produces a tagged `PrometheusMeterRegistry`, exposes a `PrometheusScrapeEndpoint`, and wires `CustomMetrics` for tenant-aware counters/timers.
- `framework/observability/build.gradle.kts`: promotes actuator starter to the main classpath so framework consumers inherit Prometheus infrastructure automatically.
- `framework/security/src/main/kotlin/com/axians/eaf/framework/security/config/SecurityFilterChainConfiguration.kt`: enforces `ROLE_eaf-admin` on `/actuator/prometheus` while leaving `/actuator/health` public as per AC2.
- `framework/observability/src/integration-test/kotlin/com/axians/eaf/framework/observability/metrics/PrometheusEndpointIntegrationTest.kt`: Spring Boot + Kotest integration verifies 401 vs 200 responses, tenant/service tags, and ensures zero Mockito/MockK usage by supplying explicit security beans.
- `products/licensing-server/src/main/resources/application.yml`: exposes `prometheus` in `management.endpoints.web.exposure.include` so the product runtime surfaces the scrape endpoint.
- `docs/architecture/component-specifications.md`: observability section now lists metrics configuration, custom metrics, and CQRS interceptors for discoverability.
