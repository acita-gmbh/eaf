# Story 7.2: Create "New Module" Generator

## Status
Ready for Review

## Story
**As a** Core Developer,
**I want** the CLI to have a `scaffold module` command,
**so that** I can create a new, empty Spring Modulith-compliant module instantly that adheres to our Foundation setup (Epic 1).

## Acceptance Criteria
1. Running `eaf scaffold module <name>` creates a new, complete Gradle sub-project (e.g., in the `products/` directory).
2. The new module's `build.gradle.kts` file is automatically configured to use the convention plugins (from Story 1.2).
3. A valid `ModuleMetadata.kt` file (required by Spring Modulith) is generated to define the new module's boundaries and name.

### Input Validation Requirements (SEC-002 Mitigation)

**Module Name Validation**:

```kotlin
class ModuleCommand : Runnable {
    @Parameters(index = "0", description = ["Module name (kebab-case, alphanumeric)"])
    private lateinit var moduleName: String

    override fun run() {
        // Validation regex: alphanumeric with hyphens, no special chars
        val validPattern = Regex("^[a-z][a-z0-9-]*$")
        require(moduleName.matches(validPattern)) {
            "Invalid module name: '$moduleName'. Must be lowercase alphanumeric with hyphens (kebab-case)."
        }

        // Path traversal protection
        require(!moduleName.contains("..")) {
            "Security violation: path traversal detected in module name"
        }

        // Template injection protection
        require(!moduleName.contains("<") && !moduleName.contains(">")) {
            "Security violation: potential injection detected in module name"
        }

        // Proceed with generation...
    }
}
```

**Attack Scenarios to Test (7.2-UNIT-006)**:
1. Path traversal: `../../etc/passwd`
2. Template injection: `<script>alert('xss')</script>`
3. Command injection: `module; rm -rf /`
4. Null bytes: `module\0.txt`
5. Unicode normalization: `\u202Emodule`

### File System Operation Safety

**Error Handling Pattern**:

```kotlin
class ModuleGenerator(private val templateEngine: TemplateEngine) {
    fun generateModule(moduleName: String, directory: Path): Either<GeneratorError, ModuleInfo> = either {
        // Check existing directory
        val modulePath = directory.resolve(moduleName)
        ensure(!Files.exists(modulePath)) {
            GeneratorError.ModuleAlreadyExists(moduleName, modulePath)
        }

        // Create directory structure
        try {
            Files.createDirectories(modulePath)
        } catch (ex: IOException) {
            return GeneratorError.FileSystemError(
                "Failed to create module directory",
                ex
            ).left()
        }

        // Generate files...
        ModuleInfo(moduleName, modulePath)
    }
}

sealed class GeneratorError {
    data class ModuleAlreadyExists(val name: String, val path: Path) : GeneratorError()
    data class FileSystemError(val message: String, val cause: Throwable) : GeneratorError()
    data class TemplateError(val template: String, val cause: Throwable) : GeneratorError()
}
```
