2. The validation chain successfully performs all 10 validation layers (Format, Algorithm (RS256-only), Signature, Claims Schema, Time (exp/iat), Issuer/Audience, Token Blacklist (stubbed), Tenant ID presence, Rate Limiting (stubbed), and Audit Logging).
3. Integration tests (using the **provisioned Keycloak Testcontainer (3.1)**) confirm that valid tokens pass all layers, and tokens with invalid signatures, expired timestamps, missing tenant claims, or wrong algorithms are correctly rejected (401).

### 10-Layer JWT Validation Architecture
The validation system implements comprehensive security through ten distinct validation layers, each addressing specific attack vectors and security concerns. [Source: architecture/security.md#10-layer-jwt-validation-system]

**Layer 1: Format Validation** - JWT structure verification, size limits, basic format checks [Source: architecture/security.md#layer-1-format-validation]
**Layer 2: Signature Validation** - Real cryptographic verification (RS256) with Keycloak public key verification [Source: architecture/security.md#layer-2-signature-validation]
**Layer 3: Algorithm Validation** - Prevent algorithm confusion attacks by enforcing RS256 only [Source: architecture/security.md#layer-3-algorithm-validation]
**Layer 4: Claim Schema Validation** - Required claims enforcement including mandatory tenant_id [Source: architecture/security.md#layer-4-claim-schema-validation]
**Layer 5: Time-based Validation** - exp/iat/nbf with clock skew tolerance (60 seconds) [Source: architecture/security.md#layer-5-time-based-validation]
**Layer 6: Issuer/Audience Validation** - Trust boundary enforcement with EAF-specific values [Source: architecture/security.md#layer-6-issuer-audience-validation]
**Layer 7: Token Revocation Check** - Redis blacklist verification (stubbed for initial implementation) [Source: architecture/security.md#layer-7-revocation-check]
**Layer 8: Role Validation** - Role whitelist and privilege escalation detection [Source: architecture/security.md#layer-8-role-validation]
**Layer 9: User Validation** - User existence, active status, account expiration [Source: architecture/security.md#layer-9-user-validation]
**Layer 10: Injection Detection** - SQL injection, XSS, JNDI attack patterns [Source: architecture/security.md#layer-10-injection-detection]
