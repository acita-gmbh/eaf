## Story
**As a** Core Developer,
**I want** the `CreateWidgetCommand` to be handled and the resulting `WidgetCreatedEvent` to be successfully persisted,
**so that** I can prove the "write-side" of CQRS and the native Axon-PostgreSQL adapter (Strategy A) are working correctly.

## Acceptance Criteria
1. The Axon Framework is configured to use the `JpaEventStorageEngine` (or `JdbcEventStorageEngine`) targeting the PostgreSQL instance launched via Docker Compose (from Story 1.3).
2. A basic API endpoint (e.g., `POST /widgets`) exists to dispatch the `CreateWidgetCommand`.
3. An integration test (using Kotest and Testcontainers, per NFR8) successfully dispatches the command.
4. The integration test verifies (by querying the Axon `domain_event_entry` table directly in the Testcontainer) that the `WidgetCreatedEvent` was correctly persisted with the proper aggregate ID and sequence number.
