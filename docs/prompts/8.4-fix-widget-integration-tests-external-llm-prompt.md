## Specific Problem Details

### jOOQ Table Reference
Generated constant in `jooq/tables/references/Tables.kt`:
```kotlin
val WIDGET_PROJECTION: WidgetProjection = WidgetProjection.WIDGET_PROJECTION
```

Repository usage:
```kotlin
dsl.selectFrom(WIDGET_PROJECTION)  // Generates SQL: SELECT ... FROM "WIDGET_PROJECTION"
```

### PostgreSQL Behavior
- Unquoted identifiers: Converted to lowercase (`widget_projection`)
- Quoted identifiers: Case-preserved (`"WIDGET_PROJECTION"`)
- Current DDL creates: `CREATE TABLE widget_projection` (lowercase)
- jOOQ queries: `SELECT ... FROM "WIDGET_PROJECTION"` (uppercase, quoted)
- Result: Mismatch → "relation does not exist"

## Possible Solutions to Investigate

### Option A: Configure jOOQ to Generate Lowercase
- Modify `build.gradle.kts` jooq configuration
- Add generator strategy to override table naming
- Force lowercase or unquoted identifiers
- **Risk**: Unknown if jOOQ supports this with DDLDatabase

### Option B: Make PostgreSQL Use Uppercase
- Modify DDL to use quoted identifiers: `CREATE TABLE "WIDGET_PROJECTION"`
- Ensures case preservation in PostgreSQL
- **Risk**: DDL file is shared with jOOQ generation (circular dependency?)

### Option C: Configure jOOQ RenderNameCase
- Use `org.jooq.conf.Settings` with `RenderNameCase.LOWER`
- Apply via `DSLContext` configuration in test
- **Risk**: Might affect generated code expectations

### Option D: Custom Table Name Strategy
- Override jOOQ's table naming in generated code
- Use @Table annotation or custom generator strategy
- **Risk**: Complex, might break code generation

### Option E: Different Schema Initialization Approach
- Use Flyway for test schema (heavyweight but proven)
- Use jOOQ's built-in DDL execution
- Create table via JDBC before Spring Boot context
- **Risk**: Timing complexity with Testcontainers + Spring Boot
