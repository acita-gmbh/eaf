# Story 4.4: E2E Test Migration Note

## E2E Test Location Decision

The `TenantContextPropagationE2EIntegrationTest` was initially created in `framework/widget/src/integration-test/` but has been **removed from framework module** for the following architectural reasons:

### Why E2E Tests Don't Belong in Framework Modules

**Framework modules are LIBRARIES**, not runnable applications:
- Cannot provide full Spring Boot application context
- No real DataSource (would require product-specific configuration)
- No JPA entity scanning (product-specific entities)
- No RLS schema (product-specific database setup)
- No Axon event store configuration (product-specific)

### Proper Test Layering

**Framework-Level Tests** (✅ Implemented in Story 4.4):
- Unit tests: Interceptor logic, metadata extraction, fail-closed validation
- Integration tests: Thread safety with Redis Testcontainers
- Component tests: Individual framework components with minimal dependencies

**Product-Level E2E Tests** (⏳ To be implemented):
- Full CQRS flow: Command → Event → Async Projection
- Real database: PostgreSQL with RLS policies
- Complete infrastructure: Axon event store, tracking processors
- Product context: Full Spring Boot application with all beans

### Migration Plan for E2E Test

// Test validates:

### Why This Is Correct

**Industry Pattern**: Spring Data, Spring Security test at library level with:
- @DataJpaTest (sliced context, no full app)
- Unit tests with test fixtures
- Integration tests with minimal beans

**Full E2E tests** requiring complete application context are at:
- Sample projects (spring-boot-samples)
- Reference implementations
- Product applications
